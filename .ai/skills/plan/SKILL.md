# Skill: plan

이 스킬은 개발의 모든 단계에서 **투명한 기록과 안전한 절차**를 보장합니다.

## 🚨 [MANDATORY] 개발 워크플로우 (절대 생략 금지)

### A. 초기 계획 단계 (큰 Task -> 세부 태스크 분할)
Phase 내의 특정 기능을 처음 시작할 때 수행합니다.
1. **[텍스트 계획 제안]**: `write_file` 도구를 사용하지 않고, 채팅창에 **텍스트로만** 세부 단계를 제안합니다.
2. **[사용자 승인]**: 사용자의 명시적인 승인("진행해", "좋아" 등)을 확인합니다.
3. **[TODO.md 기록]**: 승인된 후에만 `write_file`을 사용하여 `TODO.md`에 기록합니다.

### B. 세부 태스크 실행 단계 (TODO.md 항목별 진행)
1. **[B-1. 태스크 계획 제안]**: `TODO.md` 항목 선정 후 **구현 논리/전략** 설명 (**코드 블록 포함 절대 금지**).
2. **[B-2. 전체 코드 제안]**: 전략 승인 시, **수정될 전체 소스 코드(Full Code)**를 채팅창에 먼저 제안.
3. **[B-3. 최종 승인 및 반영]**: 코드 전문 최종 승인 후에만 `replace`, `write_file` 도구 호출 (성급한 반영 금지).

### C. 사후 처리 단계 (세부 태스크 완료 후 필수 수행)
세부 태스크 한 항목이 완료될 때마다 다음 순서를 무조건 지킵니다. (커밋 전 다음 세부 태스크 금지)
1. **[리포트 작성]**: `.person/reports/YYYY-MM-DD/{기능명}-report.md` 작성 (폴더 자동 생성).
    - 양식: 제목(#) 바로 아래에 날짜, 작성자(git config user.name 결과), 작업 단계 명시.
2. **[상태 동기화]**: `TODO.md`와 `.person/docs/phases.md`의 해당 항목을 체크(`[x]`)합니다.
3. **[자동 커밋]**: 한글 커밋 메시지를 제안하고, 승인 후 `git commit`을 실행합니다.

## 🛠️ 핵심 지침 (Core Directives)

- **코드 품질**: "기존 코드와 동일"과 같은 생략 없이 **전체 소스 코드를 제공**하고 주석을 포함합니다.
- **맥락 독립성**: `TODO.md` 작성 시 파일 경로와 핵심 로직을 상세히 적어 이전 대화 없이도 수행 가능하게 합니다.
- **작업 재개**: 세션 시작 시 `TODO.md`가 존재한다면 첫 번째 미완료(`[ ]`) 항목을 확인하고 승인을 얻어 **B-1 단계**부터 시작합니다.
  - **[TIP]** `TODO.md`가 `.gitignore`에 등록되어 있어 도구(`read_file`, `glob`)에서 찾지 못할 경우, `run_shell_command`(`type TODO.md` 또는 `cat TODO.md`)를 사용하거나 도구의 `no_ignore` (또는 `respect_git_ignore: false`) 옵션을 활성화하여 강제로 읽어야 합니다.
- **수동적 할 일 선정**: `TODO.md`가 없거나 모든 항목이 완료된 경우, `phases.md`를 근거로 다음 할 일을 스스로 제안하지 않습니다. 반드시 사용자가 새로운 기능 구현을 요청할 때까지 대기합니다.
