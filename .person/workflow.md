# 🛣️ MVP 개발 워크플로우 (Development Roadmap)

본 문서는 캠핑장 예약 시스템의 MVP(Minimum Viable Product) 단계별 개발 계획과 우선순위를 정의합니다. 모든 개발 작업은 아래 페이즈 순서에 따라 순차적으로 진행됩니다.

---

## 🏗️ Phase 1: 도메인 설계 및 환경 구축 (25%)
**목표:** 견고한 기술적 토대 마련 및 데이터베이스 구조 확립

- [x] **프로젝트 초기화:** 패키지 구조(`com.camping.erp`), 기본 폴더링(`domain`, `global`, `layout`) 완료
- [x] **규칙 수립:** `common-rule`, `business-rule`, `AI-CONTEXT` 기반의 협업 가이드 확정
- [x] **데이터베이스 ERD 설계:** 
  - `User` (회원/관리자)
  - `Zone` & `Site` (캠핑 구역 및 요금)
  - `Reservation` (예약 및 상태)
  - `Payment` (결제 및 환불)
  - `Board` (공지, 갤러리, Q&A)
  - `Review` (이용 완료자 전용)
  - `Image` (다중 이미지 관리)
- [ ] **JPA Entity 매핑:** 도메인별 Entity 클래스 작성 및 연관관계(1:N 등) 설정
- [ ] **데이터 검증:** `testData.sql`을 통한 초기 데이터 로드 및 JPA Repository 기본 동작 확인

## 🔑 Phase 2: 핵심 비즈니스 로직 및 사용자 플로우 (50%)
**목표:** 예약 및 결제라는 서비스의 '핵심 가치' 구현

- [ ] **인증 체계:** 회원가입/로그인 구현 및 `LoginInterceptor`를 통한 권한 인가 처리
- [ ] **조회 로직:** 메인 페이지 및 날짜별 예약 가능 사이트 실시간 렌더링 (SSR)
- [ ] **예약 선점(Lock):** 결제 진입 시 5분간 사이트 점유(Reservation Lock) 로직 구현
- [ ] **결제 연동:** 포트원(PortOne) V2 API 연동, 서버사이드 결제 검증 및 '즉시 확정' 처리

## 💬 Phase 3: 예약 관리 및 커뮤니티 (75%)
**목표:** 사용자 편의 기능 및 소통 창구 활성화

- [ ] **마이페이지:** 본인의 예약 내역 확인, 상세 정보 조회 및 취소 '요청' 기능
- [ ] **커뮤니티 게시판:** 
  - 공지사항 및 포토 갤러리 (조회 위주)
  - Q&A 게시판 (작성 및 관리자 답변 대기)
- [ ] **리뷰 시스템:** '이용 완료' 상태 예약 건에 대한 별점 및 사진 리뷰 작성 기능

## 🛡️ Phase 4: 관리자 모드 및 서비스 고도화 (100%)
**목표:** 운영 효율화 및 서비스 안정성 확보

- [ ] **관리자 대시보드:** 실시간 예약 현황 모니터링, 매출/예약 통계 시각화
- [ ] **운영 제어:** 
  - 취소 요청 건에 대한 위약금 자동 계산 및 '최종 승인'을 통한 환불 연동
  - 시즌별(성수기/비수기) 요금 정책 유연화
- [ ] **예외 처리 & 안정화:** 
  - `GlobalExceptionHandler`를 통한 전역 예외 처리 고도화
  - 통합 테스트(Integration Test) 및 동시성 제어(Race Condition) 최종 점검
- [ ] **배포 준비:** 빌드 환경 최적화 및 최종 검수

---

## 🛠️ 개발 원칙 (Core Principles)
1. **SSR 우선:** 모든 화면은 Mustache 템플릿 엔진을 사용하는 서버 사이드 렌더링 방식으로 개발한다.
2. **Form 기반:** 데이터 변경은 `<form method="POST">`와 PRG 패턴을 준수한다.
3. **Surgical Update:** 모든 수정은 기존 규칙(AI-CONTEXT, Common Rule)을 엄격히 따르며 기록한다.
