# 리포트: 위약금 자동 산출 및 부분 환불 시스템 구현

**Date**: 2026-03-21  
**Reporter**: Gemini CLI  
**Task Phase**: Phase 3-2.4 (Task 8)

## 📝 작업 요약
체크인 날짜를 기준으로 위약금을 자동으로 계산하고, 포트원 API와 연동하여 실결제를 부분 취소하는 통합 환불 시스템을 구축했습니다.

## 🛠️ 변경 사항
1. **위약금 계산 엔진 (`RefundService`)**:
    - 비즈니스 규칙에 따라 체크인 기준 남은 일수를 계산하여 환불 금액을 자동 산출합니다.
    - **7일 전**: 100% 환불
    - **3~6일 전**: 50% 환불 (부분 취소)
    - **0~2일 전**: 자동 환불 불가 (시스템 제한)
2. **포트원 부분 취소 연동**:
    - `PortOneService` 인터페이스에 `amount` 파라미터를 추가하여, 계산된 환불 금액만큼만 실제로 카드 결제가 취소되도록 구현했습니다.
3. **관리자 환불 승인 프로세스**:
    - 관리자 대시보드에서 환불 요청 승인 시, 실시간으로 위약금 규정을 재검증하고 포트원 취소 요청을 보냅니다.
    - 취소가 완료되면 예약 상태를 `CANCEL_COMP`로 변경하고 `Refund` 이력을 투명하게 저장합니다.
4. **안전 장치 도입**:
    - 이용 당일~2일 전 예약에 대해서는 시스템을 통한 자동 환불 승인을 차단하고 고객센터 유선 연락을 유도하는 방어 로직을 강화했습니다.

## 💡 비유로 설명하기
이 작업은 **"자동 정산기가 있는 주차장"**과 같습니다.
- **위약금 계산**: 주차 시간에 따라 요금이 달라지듯, 체크인까지 남은 시간에 따라 돌려받을 돈이 자동으로 정산됩니다.
- **부분 환불**: 만원 결제 후 5천원어치만 서비스를 이용했다면, 결제 전체를 취소하고 다시 긁는 번거로움 없이 5천원만 '부분 취소' 해주는 편리한 기능입니다.
- **시스템 제한**: 주차 차단기가 고장 났을 때 무리하게 열지 않고 '관리자 호출' 버튼을 누르게 하듯, 이용 직전 취소는 반드시 사람(고객센터)을 거치게 하여 사고를 방지합니다.

- 포트원 Mock 서비스를 통한 부분 환불 요청 정상 발송 확인.
- `Refund` 테이블 이력 저장 및 예약 상태 전이 확인.
