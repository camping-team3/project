# 리포트: 결제 페이지 고도화 및 이탈 감지 로직 구현

**Date**: 2026-03-21  
**Reporter**: Gemini CLI  
**Task Phase**: Phase 3-2.2 (Task 5)

## 📝 작업 요약
PortOne V2 SDK 연동을 완료하고, 사용자가 결제 도중 이탈할 경우 선점된 자원을 즉시 해제하는 지능형 스크립트를 구현했습니다.

## 🛠️ 변경 사항
1. **PortOne V2 SDK 통합**:
    - 결제 페이지에 PortOne 브라우저 SDK를 로드하고, `storeId` 및 `channelKey`를 서버로부터 안전하게 전달받아 결제 요청 기능을 구현했습니다.
2. **이탈 감지 및 자동 해제 (Smart Release)**:
    - `beforeunload` 이벤트를 활용하여 브라우저 종료, 새로고침, 뒤로가기 상황을 감지합니다.
    - `fetch` API의 `keepalive: true` 옵션을 적용하여, 브라우저가 닫히는 찰나에도 서버에 '선점 해제(DELETE)' 요청이 반드시 도달하도록 설계했습니다.
    - 이를 통해 사용자가 '뒤로가기'를 누르면 방금 보던 사이트가 목록에 즉시 다시 나타나도록 UX를 개선했습니다.
3. **결제 프로세스 보호**:
    - `isProcessingPayment` 플래그를 도입하여, 실제 결제가 진행 중인 시점(PortOne 창 활성화 등)에는 이탈 감지 로직이 자원을 해제하지 않도록 방어 로직을 추가했습니다.

## 💡 비유로 설명하기
이 작업은 **"뷔페 예약석 팻말"**과 같습니다.
- **가예약**: 손님이 자리에 앉으려 할 때 '예약석' 팻말을 놓습니다.
- **이탈 감지**: 손님이 주문하지 않고 식당 문을 나가는 순간, 종업원이 번개같이 달려가 팻말을 치우고 다음 손님이 앉을 수 있게 하는 것과 같습니다. `keepalive` 옵션은 종업원이 문이 닫히기 전까지 끝까지 달려가는 끈기와 같습니다.

## ✅ 검증 결과
- 결제 버튼 클릭 시 PortOne SDK 호출 확인.
- 브라우저 종료/뒤로가기 시 `/api/reservations/{id}/lock` API 호출 확인.
