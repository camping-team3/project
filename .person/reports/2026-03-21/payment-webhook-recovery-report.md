# 리포트: 결제 검증 웹훅 및 조건부 자동 복구 시스템 구현

**Date**: 2026-03-21  
**Reporter**: Gemini CLI  
**Task Phase**: Phase 3-2.3 (Task 6, 7)

## 📝 작업 요약
포트원 결제 완료 신호를 처리하는 웹훅 시스템을 구축하고, 데이터 유실 시에도 안전하게 예약을 복원하는 **'조건부 자동 복구'** 로직을 도입했습니다.

## 🛠️ 변경 사항
1. **결제 검증 웹훅 (`POST /api/payment/webhook`)**:
    - 포트원으로부터 결제 성공 신호를 수신하여 서버 측에서 최종 결제 금액을 검증합니다.
    - 금액 위변조가 감지될 경우 즉시 자동 환불을 실행하여 보안을 강화했습니다.
2. **조건부 자동 복구 (Conditional Recovery)**:
    - 선점 시간(10분) 만료로 인해 DB에서 예약 데이터가 이미 삭제된 상태에서 결제 신호가 온 경우를 대비했습니다.
    - 해당 시점에 사이트가 여전히 비어 있다면 정보를 재생성하여 예약을 **자동 복구**합니다.
3. **오버부킹 방지 및 자동 환불**:
    - 복구 시점에 이미 다른 사용자가 해당 자리를 예약했다면(오버부킹), 시스템이 이를 감지하고 포트원 취소 API를 호출하여 고객에게 **즉시 자동 환불**을 진행합니다.
4. **Mock 통신 레이어**:
    - 외부 API 의존성을 낮추기 위해 `PortOneService` 인터페이스와 Mock 구현체를 사용하여 로직의 흐름을 완성했습니다.

## 💡 비유로 설명하기
이 작업은 **"식당 예약석과 키오스크"**의 관계와 같습니다.
- **웹훅**: 키오스크에서 결제가 끝나면 주방에 '주문서'가 전달되는 것과 같습니다.
- **자동 복구**: 만약 주방장이 주문서를 받기 직전에 '노쇼'인 줄 알고 자리를 치웠는데, 실제로는 손님이 결제를 마친 상태라면? 주방장은 재빨리 자리가 비었는지 보고 다시 자리를 마련해줍니다.
- **자동 환불**: 하지만 그 0.1초 사이에 다른 손님이 앉아버렸다면, 어쩔 수 없이 결제된 돈을 정중히 돌려드리는 시스템입니다.

## ✅ 검증 결과
- 웹훅 수신 시 `PaymentService` 로직 정상 작동 확인.
- 데이터 삭제 시나리오에서 가용성 체크 후 조건부 복구/환불 로직 검증 완료.
- 리포트에 요청하신 복구 로직 상세 내용 포함 완료.
