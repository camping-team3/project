# 리포트: 가예약(Lock) 생성 및 즉시 해제 API 구현

**Date**: 2026-03-21  
**Reporter**: Gemini CLI  
**Task Phase**: Phase 3-2.2 (Task 4)

## 📝 작업 요약
결제 프로세스의 안정성과 자원 선점의 효율성을 높이기 위해 가예약(PENDING) 생성 로직과 즉시 해제 API를 구현했습니다.

## 🛠️ 변경 사항
1. **가예약 생성 로직 도입**:
    - 기존에 정보 조회만 수행하던 결제 페이지 진입 단계에서 실제로 `PENDING` 상태의 `Reservation` 데이터를 생성하도록 변경했습니다.
    - 이를 통해 사용자가 결제 페이지에 머무르는 동안 다른 사용자가 동일한 사이트를 예약할 수 없도록 물리적인 'Lock'을 설정했습니다.
2. **선점 즉시 해제 API 구현**:
    - `DELETE /api/reservations/{id}/lock` 엔드포인트를 추가했습니다.
    - 사용자가 결제창을 이탈할 때 이 API를 호출하여, 불필요하게 선점된 자원을 즉시 해제(물리적 삭제)할 수 있도록 했습니다.
    - 보안을 위해 세션 유저와 예약자가 일치하는 경우에만 해제가 가능하도록 검증 로직을 포함했습니다.

## 💡 비유로 설명하기
이 작업은 마치 **"인기 있는 식당에서 줄을 서는 것"**과 같습니다.
- **기존 방식**: 메뉴판만 보고 있다가 실제 주문을 해야 자리가 확보됨. (주문 직전 자리가 뺏길 수 있음)
- **개선 방식**: 메뉴판을 받는 순간 일단 자리에 앉히는 것(PENDING 예약). 만약 손님이 주문하지 않고 그냥 나가면 즉시 자리를 치우고 다음 손님을 앉힙니다(즉시 해제 API).

## ✅ 검증 결과
- `ReservationService` 유닛 테스트 및 컨트롤러 매핑 확인 완료.
- 세션 유저 불일치 시 `403 Forbidden` 응답 확인.
