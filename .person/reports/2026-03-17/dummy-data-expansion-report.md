## 📝 작업 요약
페이징 기능 검증을 위해 주요 테이블(`user_tb`, `notice_tb`, `gallery_tb`, `qna_tb`, `review_tb`, `reservation_tb`)의 더미 데이터를 대폭 확충하였습니다.

## 🛠️ 변경 사항
- `src/main/resources/db/data.sql`: 각 테이블당 최소 20건 이상의 데이터를 추가로 삽입하는 SQL 구문 작성 및 반영.
- H2 데이터베이스 호환성을 위해 `DATEADD` 함수를 사용하여 생성 시간을 역순으로 배치, 페이징 정렬 순서 확인이 가능하도록 설계.

## ✅ 검증 결과
- **사용자**: 6명 -> 21명 (관리자 포함)
- **공지사항**: 3건 -> 23건
- **포토 갤러리**: 2건 -> 22건
- **Q&A**: 2건 -> 22건
- **이용 후기**: 1건 -> 21건
- **예약 이력**: 5건 -> 25건
- 모든 게시판에서 페이지 당 10개 출력 기준 3페이지까지 노출되는 것을 기대할 수 있음.

## 💡 비유로 설명하기
마치 **새로 지은 도서관에 책이 몇 권 없어서 대출 시스템(페이징)이 잘 작동하는지 알 수 없었던 상황**이었는데, 이번 작업을 통해 **수천 권의 책(더미 데이터)을 서가에 꽂아둠으로써** 이제는 독자들이 다음 페이지로 넘겨가며 책을 잘 찾을 수 있는지 확실히 테스트해 볼 수 있게 된 것과 같습니다.
