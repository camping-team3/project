# [공지사항] 페이징 UI 구현 보고서

## 1. 개요
공지사항 목록 페이지(관리자/사용자)에서 데이터가 10개를 초과할 때 페이지를 이동할 수 있는 내비게이션 UI를 구현하였습니다.

## 2. 주요 변경 사항
- **global/dto/PageResponse.java (신규)**:
    - Spring Data JPA의 0-based 페이지 번호를 1-based로 변환.
    - Mustache 템플릿에서 직접 사용 가능한 `pageNumbers` 리스트(현재 페이지 활성화 정보 포함) 생성.
    - 이전/다음 페이지의 존재 여부 및 대상 인덱스 계산 로직 통합.
- **AdminNoticeController / NoticeController**:
    - `Page<T>` 대신 `PageResponse<T>`를 모델에 담아 뷰로 전달하도록 수정.
- **Mustache 템플릿 (admin/notice/list, notice/list)**:
    - 1부터 시작하는 페이지 번호 버튼 렌더링.
    - 현재 페이지 하이라이트(`active`) 및 버튼 크기 고정(32px/40px).
    - 이전/다음 버튼 비활성화 상태 시각적 피드백 제공.
    - 페이지 이동 시 검색어(`keyword`)가 유지되도록 URL 파라미터 연동.

## 3. 검증 결과
- 공지사항이 10개 이상일 때 페이징 번호(1, 2...)가 정상적으로 노출됨.
- 페이지 번호 클릭 시 해당 페이지로 데이터가 올바르게 갱신됨.
- 검색 후 페이지 이동 시 검색 결과 내에서 페이징이 유지됨을 확인.

## 4. 비유로 설명하기
이전의 페이징은 마치 **두꺼운 책에 목차가 없어서 다음 장으로 넘기기만 가능하고, 내가 지금 몇 페이지를 읽고 있는지 알기 어려운 상태**였습니다.

이번 수정을 통해, 책 하단에 **정확한 페이지 번호(1, 2, 3...)와 내가 지금 보고 있는 페이지를 표시하는 북마크**를 만들었습니다. 이제 독자(사용자)는 원하는 페이지로 바로 건너뛸 수 있고, 더 이상 읽을 페이지가 없을 때는 '다음' 버튼이 회색으로 변해 더 넘길 수 없음을 직관적으로 알 수 있게 되었습니다.
