# [공지사항] 수정 시 이미지 프리뷰 버그 수정 및 삭제 기능 구현 보고서

## 1. 개요
공지사항 수정 화면에서 새로운 이미지를 추가했을 때 화면에 보이지 않던 버그를 수정하고, 기존에 등록된 이미지를 개별적으로 삭제할 수 있는 기능을 추가하였습니다.

## 2. 주요 변경 사항
- **update-form.mustache (프론트엔드)**:
    - **버그 수정**: JavaScript의 `FileReader` 호출 시 발생했던 오타(`readAsAsDataURL`)를 `readAsDataURL`로 수정하여 새로운 이미지 추가 시 즉시 프리뷰가 나타나도록 개선.
    - **삭제 UI 추가**: 기존 이미지 우측 상단에 삭제 버튼(`×`) 추가.
    - **삭제 데이터 관리**: 기존 이미지 삭제 시 해당 이미지의 ID를 `hidden input`(`name="deleteImageIds"`)으로 동적 생성하여 폼 제출 시 서버에 전달.
- **NoticeRequest & NoticeService (백엔드)**:
    - **UpdateDTO**: 삭제할 이미지 ID 목록(`List<Long> deleteImageIds`) 필드 추가.
    - **NoticeService.update()**: 전달받은 `deleteImageIds`가 있을 경우, `Notice` 엔티티의 `images` 리스트에서 해당 ID를 가진 객체를 제거하여 `orphanRemoval` 옵션을 통해 DB에서 자동 삭제되도록 처리.

## 3. 검증 결과
- 수정 화면에서 이미지를 추가하면 "신규" 태그와 함께 즉시 화면에 나타남을 확인.
- 기존 이미지의 '×' 버튼 클릭 시 화면에서 사라지며, 수정 완료 후 해당 이미지가 실제 데이터베이스에서 삭제됨을 확인.
- 제목/내용 수정과 이미지 추가/삭제가 한 번의 요청으로 조화롭게 작동함을 확인.

## 4. 비유로 설명하기
이전의 수정 기능은 마치 **이미 적어둔 편지에 새 사진을 붙이려고 가져왔는데 사진이 투명해져서 안 보이고, 원래 붙어있던 사진을 떼어내고 싶은데 본드로 너무 강력하게 붙어서 뗄 수 없는 상황**이었습니다.

이번 수정을 통해, **가져온 사진이 선명하게 보이게 고쳤고(프리뷰 버그 수정), 원래 붙어있던 사진에는 '떼기' 버튼을 달아주어 언제든 원할 때 떼어낼 수 있게(기존 이미지 삭제 기능) 만들었습니다.** 이제 사용자는 편지를 완벽하게 재구성할 수 있습니다.
