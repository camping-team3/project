# 📝 Q&A 및 답변 시스템 구현 보고서

## 1. 작업 요약
Phase 3의 핵심 도메인인 **Qna & Comment (질의응답 및 답변)** 기능을 구현하였습니다. 사용자의 자유로운 문의 등록과 관리자의 신속한 답변 처리를 지원하며, 답변이 완료된 문의에 대한 데이터 무결성을 보장하는 비즈니스 로직을 적용했습니다.

## 2. 주요 변경 사항

### 2.1 데이터베이스 스키마 및 초기화 데이터 변경
- **컬럼 추가**: `qna_tb` 테이블에 `category`(Enum)와 `hits`(Integer) 컬럼을 추가하고, 기존 `ERD.md` 명세를 최신화했습니다.
- **초기 데이터 정합성 해결**: `category` 컬럼에 `NOT NULL` 제약 조건이 추가됨에 따라, `src/main/resources/db/data.sql`의 기존 INSERT 문에서 발생하던 무결성 제약 조건 위반 에러를 해결했습니다 (샘플 데이터에 카테고리 값 할당).

### 2.2 도메인 모델 고도화
- **QnaCategory Enum**: 문의 유형을 `RESERVATION`(예약), `FACILITY`(시설), `ETC`(기타)로 체계화했습니다.
- **Qna 엔티티 확장**: 조회수(`hits`), 카테고리(`category`), 답변여부(`isAnswered`) 필드를 추가하고, 캡슐화된 상태 변경 메서드를 구현했습니다.
- **Comment 엔티티 연동**: 관리자 정보(`User admin`)와 연관관계를 맺어 답변 주체를 명확히 했습니다.

### 2.2 비즈니스 로직 및 규칙 적용
- **수정 제한 (Immutable after Answer)**: 답변이 완료된(`isAnswered=true`) 질문은 작성자가 내용을 수정할 수 없도록 차단했습니다 (`Exception400` 발생).
- **자동 상태 전이**: 관리자가 답변을 등록하면 해당 질문의 상태가 즉시 `답변완료`로 전환됩니다.
- **조회수 로직**: 상세 페이지 접근 시 조회수가 자동으로 증가하도록 처리했습니다.

### 2.3 UI/UX 및 컨트롤러 연동
- **고객 사이드**: 목록 조회, 질문 작성/수정/삭제 기능을 Mustache 템플릿과 연동했습니다.
- **관리자 사이드**: 전체 Q&A 현황 파악 및 상세 답변 등록 인터페이스를 구축했습니다.
- **SSR & Form**: 모든 데이터 변경은 POST 요청과 PRG(Post-Redirect-Get) 패턴을 준수하여 구현했습니다.

## 3. 검증 결과
- **정상 케이스**: 로그인 사용자의 질문 등록, 목록 조회, 관리자 답변 등록 및 상태 변경 확인.
- **예외 케이스**:
    - 본인이 아닌 사용자의 수정/삭제 시도 시 `Exception403` 확인.
    - 답변 완료된 글 수정 시도 시 비즈니스 규칙에 따른 차단 확인.

## 4. 비유로 보는 Q&A 시스템
이 시스템은 **'도장이 찍힌 봉투'**와 같습니다.
처음에 사용자가 문의글을 써서 봉투에 넣으면 언제든지 내용을 고칠 수 있습니다. 하지만 관리자가 확인 후 **'답변 도장(Answered)'**을 쾅! 찍는 순간, 이 봉투는 공식적인 기록이 되어 내용을 고칠 수 없게 봉인됩니다. 이는 나중에 사용자가 질문 내용을 교묘하게 바꿔 답변의 맥락을 왜곡하는 것을 방지하기 위한 장치입니다.
