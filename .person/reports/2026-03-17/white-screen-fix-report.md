## 📝 작업 요약
홈페이지 백지 화면 출력 문제를 해결하기 위해 `GlobalExceptionHandler`의 응답 방식을 수정하고, `data.sql`의 데이터 구조를 ID 명시적 방식으로 개선하였습니다.

## 🛠️ 변경 사항
- **GlobalExceptionHandler 수정**:
    - `@RestControllerAdvice`를 `@ControllerAdvice`로 변경하여 HTML/Script 응답이 JSON 문자열로 직렬화되는 문제를 해결하였습니다.
    - 브라우저가 예외 발생 시 스크립트를 즉시 실행하여 사용자에게 에러 메시지를 노출하도록 조치하였습니다.
- **data.sql 정적 ID 부여**:
    - 모든 기초 테이블(`user_tb`, `zone_tb`, `site_tb`)에 고정 ID를 부여하여, 자동 증가 ID 값의 불확실성으로 인한 참조 오류(FK Violation) 및 NullPointerException을 방지하였습니다.
    - 테이블 간의 연관관계를 명확히 하여 DTO 변환 시 데이터 누락으로 인한 렌더링 실패를 차단하였습니다.

## ✅ 검증 결과
- **정상 렌더링**: 홈페이지 접속 시 `sites` 데이터가 정상적으로 바인딩되어 화면이 출력됩니다.
- **예외 처리 검증**: 고의적으로 에러를 유발했을 때 백지가 아닌 `alert`창과 함께 이전 페이지로 돌아가는 정상적인 예외 흐름을 확인하였습니다.
- **데이터 정합성**: 모든 사이트가 자신이 속한 구역 정보를 정확히 참조하여 가격 및 평점이 정상적으로 표시됩니다.

## 💡 비유로 설명하기
마치 **전기 회로(애플리케이션)가 합선되어 차단기(GlobalExceptionHandler)가 내려갔는데, 정작 차단기 박스가 잠겨 있어 집 안이 깜깜(백지 화면)했던 상황**이었습니다. 이번 작업을 통해 차단기 박스를 열어 에러를 확인할 수 있게 고치고, 애초에 합선이 일어나지 않도록 전선(데이터 ID)들을 규격에 맞춰 단단히 고정한 것과 같습니다.
