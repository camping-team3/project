# User 도메인 예외 처리 강화 리포트

**Date:** 2026-03-17  
**Reporter:** handikim  
**Task:** UserService 예외 타입을 Exception400으로 구체화하여 사용자 알림 강화

## 작업 요약
범용적인 `RuntimeException` 대신 프로젝트에서 정의한 사용자 정의 예외인 `Exception400`을 사용하도록 수정하여, 클라이언트에게 구체적인 에러 메시지를 전달할 수 있도록 개선했습니다.

## 변경 사항
1. **UserService.java**: 모든 비즈니스 로직 오류 발생 시 던지는 예외를 `RuntimeException`에서 `Exception400`으로 변경했습니다.
   - 비밀번호 불일치, 아이디 중복, 사용자 미검색 등.
2. **GlobalExceptionHandler 연동**: `GlobalExceptionHandler`가 `Exception400`을 잡아 브라우저에서 `alert(message)`와 `history.back()`을 실행하도록 구성되어 있어, 가입이나 로그인 실패 시 사용자 경험이 개선되었습니다.

## 검증 결과
- `GlobalExceptionHandler`의 `ex400` 메서드가 정상적으로 동작하여, 서버 에러(`500`)가 아닌 클라이언트 에러(`400`)로 처리됨을 확인했습니다.
- 사용자는 오류 발생 시 팝업 창을 통해 정확한 이유를 알 수 있고, 작성 중이던 폼 데이터가 유지된 채로 이전 페이지로 돌아가게 됩니다.

## 비유로 설명
기존에는 문제가 생기면 단순히 "문이 고장 났어요"라고 퉁쳐서 말하던 안내원을, 이제는 "비밀번호가 틀렸어요" 혹은 "이미 사용 중인 이름이에요"라고 친절하고 정확하게 알려주는 안내원으로 교체한 것과 같습니다.
