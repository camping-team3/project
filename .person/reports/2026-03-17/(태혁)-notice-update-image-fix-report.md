# [공지사항] 수정 시 이미지 처리 오류 수정 보고서

## 1. 개요
공지사항 수정 화면에서 기존 이미지가 표시되지 않고, 새로운 이미지를 추가로 업로드할 수 없었던 문제를 해결하였습니다.

## 2. 주요 변경 사항
- **NoticeRequest.UpdateDTO**: 수정 요청 시 이미지 파일(`List<MultipartFile> images`)을 받을 수 있도록 필드 추가.
- **NoticeService.update()**:
    - 이미지 저장 로직을 `saveNoticeImages` 공통 메소드로 추출하여 중복 제거.
    - 수정 시에도 새로운 이미지 파일을 전달받으면 서버에 저장하고 엔티티와 연관관계를 설정하도록 로직 보완.
- **update-form.mustache**:
    - `<form>` 태그에 `enctype="multipart/form-data"` 추가하여 파일 전송 지원.
    - 서버에서 전달받은 기존 이미지 목록(`{{#notice.images}}`)을 "기존" 태그와 함께 프리뷰 그리드에 렌더링.
    - `save-form`에서 검증된 드래그 앤 드롭 및 클릭 업로드 UI를 그대로 이식하여 일관된 사용자 경험 제공.
    - 새로운 이미지 선택 시 "신규" 태그를 부여하여 기존 이미지와 시각적으로 구분.

## 3. 검증 결과
- 수정 화면 진입 시 기존에 등록했던 이미지가 썸네일로 정상 노출됨을 확인.
- 수정 화면에서 새로운 이미지를 드래그하여 추가하면 "신규" 태그와 함께 프리뷰에 추가됨을 확인.
- '수정 완료' 클릭 시 제목, 내용뿐만 아니라 추가된 이미지도 DB 및 서버 폴더에 정상적으로 저장됨을 확인.

## 4. 비유로 설명하기
이전의 수정 기능은 마치 **이미 적어둔 편지 내용을 고치려고 하는데, 이전에 붙여두었던 사진들은 보이지 않고 새로 사진을 넣으려고 해도 풀이 없어서 붙일 수 없는 상황**이었습니다.

이번 수정을 통해, **편지를 수정할 때 이전에 붙였던 사진들을 그대로 보여주고(기존 이미지 노출), 새로운 사진을 가져오면 즉시 붙일 수 있는 강력한 풀(업로드 로직 및 필드 추가)을 제공**하게 되었습니다. 이제 사용자는 글뿐만 아니라 이미지까지 완벽하게 관리할 수 있습니다.
