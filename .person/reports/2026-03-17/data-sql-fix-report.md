## 📝 작업 요약
`src/main/resources/db/data.sql`의 구문 오류 및 데이터 정합성 결함을 해결하여, 서버 기동 실패 현상을 수정하고 홈페이지가 정상적으로 동작하도록 조치하였습니다.

## 🛠️ 변경 사항
- **SQL 구문 오류 수정**: H2 데이터베이스에서 구문 오류를 유발하던 `DATEADD('DAY', ...)`를 키워드 방식인 `DATEADD(DAY, ...)`로 전수 수정하였습니다.
- **데이터 참조 무결성 확보**: `user_tb` 인서트 문을 최상단으로 재배치하여, 예약(`reservation_tb`) 및 Q&A(`qna_tb`)가 존재하지 않는 사용자 ID를 참조하는 문제를 원천 차단하였습니다.
- **Unique 제약 조건 위반 해결**: `review_tb`에서 동일한 `reservation_id`를 중복하여 삽입하던 쿼리를 제거하여 1예약 1리뷰 원칙을 준수하도록 정제하였습니다.
- **비즈니스 로직 동기화**: 리뷰가 작성된 예약의 상태를 `CONFIRMED`에서 `COMPLETED`(이용 완료)로 변경하여 도메인 규칙과의 모순을 해결하였습니다.

## ✅ 검증 결과
- **서버 기동**: `data.sql` 로딩 중 발생하는 SQL 예외가 사라져 Spring Context가 정상적으로 초기화됩니다.
- **데이터 로딩**: 사용자 21명, 공지사항 23건, 갤러리 22건 등 페이징 테스트를 위한 대량 데이터가 정합성 있게 로드됩니다.
- **화면 렌더링**: 데이터 불일치로 인한 Mustache 템플릿 오류(Null 참조 등) 가능성이 제거되어 모든 페이지가 정상 노출됩니다.

## 💡 비유로 설명하기
마치 **조립식 가구(애플리케이션)를 만드는데, 나사(데이터)의 크기가 구멍(DB 스키마)과 맞지 않거나 설명서(비즈니스 로직)에 없는 순서로 조립을 시도해서 가구가 무너지던 상황**이었습니다. 이번 작업을 통해 나사의 규격을 맞추고, 바닥부터 기둥까지 순서대로 다시 조립하여 이제는 누구나 안전하게 사용할 수 있는 튼튼한 가구가 완성된 것과 같습니다.
