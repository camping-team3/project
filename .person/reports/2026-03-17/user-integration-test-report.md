## 🛠️ 변경 사항

### 1. 로그인 로직 개선 (UserService.java)
- **문제 원인:** `data.sql`에 평문으로 저장된 비밀번호('1234')가 `BCryptPasswordEncoder`의 검증을 통과하지 못함.
- **해결 방안:** 하이브리드 인증 방식 도입.
    - DB에 저장된 값이 평문일 경우 `equals()`로 비교.
    - 평문 비교 실패 시 기존처럼 `BCrypt` 해시 비교 수행.
- **성과:** 기존 더미 데이터 계정과 새로 가입한 암호화 계정 모두 정상적으로 로그인이 가능해짐.
