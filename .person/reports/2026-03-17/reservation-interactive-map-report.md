# 리포트: 캠핑장 배치도 인터랙티브 필터링 구현
날짜: 2026-03-17
작성자: parkcoding
작업 단계: Phase 2 - Task 4

## 📝 작업 요약
사용자가 캠핑장 전체 구조를 한눈에 파악하고, 원하는 구역의 사이트를 빠르게 찾을 수 있도록 인터랙티브 배치도 기능을 구현했습니다.

## 🛠️ 변경 사항
- **배치도 이미지 연동**: 실제 캠핑장 배치도(`camping_map.png`)를 예약 페이지 상단에 적용.
- **인터랙티브 핀 배치**:
    - CSS `%` 단위를 사용하여 반응형에서도 위치가 고정되는 구역 핀(A, B, C) 배치.
    - 핀 클릭 시 활성화 상태 시각화(색상 변경 및 확대).
- **클라이언트 사이드 필터링**:
    - JavaScript를 이용해 핀 클릭 시 해당 구역 사이트만 노출하는 로직 구현.
    - **[Troubleshooting]**: DB의 구역 이름(`zoneName`)과 필터링 값이 일치하지 않던 문제를 해결하고, CSS 충돌 방지를 위해 `d-none` 클래스 기반으로 로직을 개선하여 안정성을 확보함.
- **사용자 편의성**: 동일 핀 재클릭 시 필터가 해제되는 토글 기능 추가.

## 💡 비유로 설명하기
이번 작업은 마치 **'백화점 안내 키오스크에 층별 버튼을 단 것'**과 같습니다. 예전에는 모든 매장을 일일이 스크롤해서 찾아야 했다면, 이제는 지도 위의 'A구역' 버튼만 누르면 해당 구역의 매장들만 조명이 켜지며 강조되는 것과 같은 원리입니다. 특히 버튼의 이름표를 실제 층수와 똑같이 맞추고, 전등 스위치(d-none)를 확실하게 작동하도록 고쳐서 사용자가 헷갈리지 않고 원하는 자리를 찾을 수 있게 되었습니다.
