# User 도메인 서버 사이드 검증 강화 리포트

**Date:** 2026-03-17  
**Reporter:** handikim  
**Task:** JoinDTO 비밀번호 확인 필드 추가 및 서버 사이드 검증 로직 구현

## 작업 요약
회원가입 시 보안성과 데이터 무결성을 높이기 위해 서버 사이드에서 비밀번호 일치 여부를 한 번 더 검증하는 로직을 추가했습니다.

## 변경 사항
1. **UserRequest.java**: `JoinDTO`에 `passwordConfirm` 필드를 추가하여 클라이언트로부터 비밀번호 확인 값을 받을 수 있도록 했습니다.
2. **UserService.java**: `join()` 메서드 시작 부분에 `password`와 `passwordConfirm`이 일치하는지 확인하는 로직을 추가했습니다. 불일치 시 `RuntimeException`을 발생시켜 가입을 중단합니다.

## 검증 결과
- 프론트엔드(`join-form.mustache`)에서 이미 일치 확인을 수행하고 있지만, 이번 변경으로 API 직접 호출 등을 통한 잘못된 데이터 삽입을 원천 차단했습니다.
- 비밀번호 불일치 시 "비밀번호가 일치하지 않습니다." 메시지가 정상적으로 던져지는 것을 확인했습니다.

## 비유로 설명
문을 열 때 키(비밀번호)를 한 번만 돌리는 게 아니라, "이게 정말 맞는지 한 번 더 확인(비밀번호 확인)"하는 과정을 추가한 것과 같습니다. 또한, 이 확인 작업을 문 앞(브라우저)에서만 하는 게 아니라, 문 안쪽의 관리인(서버)이 직접 대조해보고 승인해주는 구조로 변경하여 더욱 안전해졌습니다.
