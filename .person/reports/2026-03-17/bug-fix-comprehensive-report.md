# 🛠️ Q&A 시스템 종합 버그 수정 및 UI/UX 개선 보고서

## 1. 개요
로컬 서버 테스트 중 발견된 Q&A 도메인의 총 7가지 결함(기능 오류 4건, UI/UX 미비 3건)을 전수 수정하고, 시스템의 안정성과 가독성을 확보하였습니다.

## 2. 주요 수정 내용 (7가지 이슈 해결)

### 2.1 기능 및 로직 수정 (Bug Fix)
1. **필터링 기능 정상화**: `/qna` 및 `/admin/qna` 페이지에서 '답변대기/답변완료' 쿼리 파라미터(`status`) 연동을 완료했습니다.
2. **더미 데이터 상세 조회 오류 해결**: SQL로 주입된 초기 데이터 접근 시 발생하는 지연 로딩 및 Null 이슈를 `Fetch Join`과 `Null-safe DTO` 설계를 통해 해결했습니다.
3. **리다이렉션 피드백 강화**: 삭제/수정 성공 시 단순 이동이 아닌, 브라우저 `alert` 창을 통해 사용자에게 명확한 처리 결과를 알린 후 페이지를 이동시키도록 개선했습니다.
4. **목록 누락 현상 수정**: JPA 연관 관계 조인 쿼리 최적화를 통해 특정 상태의 질문이 목록에서 사라지는 현상을 방지했습니다.

### 2.2 UI/UX 및 편의성 개선
5. **이미지 업로드 인터페이스 구현**: `qna/new` 페이지에서 이미지 영역 클릭 시 파일 탐색기가 열리도록 JS를 연동하고, 선택된 파일명을 시각적으로 표시했습니다.
6. **관리자 가독성 개선**: 관리자 목록 배지의 색상을 시인성 높은 색상(녹색/노란색)으로 교체하여 답변 여부를 즉시 파악할 수 있게 했습니다.
7. **미구현 기능 안내**: '엑셀 내보내기' 등 현재 스텁 상태인 버튼 클릭 시 '준비 중' 안내창을 띄워 시스템 오류로 오해하지 않도록 조치했습니다.

## 3. 검증 시나리오 (Verification)
- **질문 등록**: `/qna/new`에서 이미지 첨부 및 등록 확인 → `/qna` 목록 리다이렉션 확인.
- **필터링 테스트**: 전체/대기/완료 탭 클릭 시 URL 파라미터와 목록 데이터의 일치 여부 확인.
- **관리자 답변**: `/admin/qna` 상세 페이지 진입 → 답변 등록 → 상태가 '답변완료'로 변하는지 확인.
- **권한 및 보안**: 답변 완료된 글의 수정 시도 시 비즈니스 규칙에 따른 차단(400 에러 및 알림) 확인.

## 4. 비유로 보는 버그 수정
이번 작업은 **'길 안내가 잘못된 도서관'**을 수리한 것과 같습니다.
분류표(필터)가 엉망이고, 낡은 책(더미 데이터)은 열람이 안 되며, 반납(삭제)을 해도 확인증을 주지 않던 도서관에 정확한 안내 표지판을 세우고, 모든 책을 정비하며, 이용자에게 친절한 알림 시스템을 도입한 과정이었습니다.
