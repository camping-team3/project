# Report: 갤러리 및 공지사항 썸네일 즉시 반영 문제 해결

**Reporter:** Gemini CLI
**Date:** 2026-03-17

## 1. 작업 요약
갤러리 및 공지사항 게시글 작성/수정 후 썸네일 이미지가 즉시 노출되지 않고 새로고침을 해야만 보이던 현상을 해결했습니다.

## 2. 변경 사항
- **외부 업로드 디렉토리 도입**: 파일을 프로젝트의 `classpath`(src/main/resources/static/upload)가 아닌, 프로젝트 루트의 `./upload/` 디렉토리에 직접 저장하도록 변경했습니다.
- **리소스 핸들러 설정**: `WebMvcConfig`에서 `/upload/**` 요청을 외부 디렉토리(`./upload/`)와 기존 정적 리소스 경로에서 모두 찾을 수 있도록 매핑했습니다.
- **서비스 로직 수정**: `GalleryService`와 `NoticeService`에서 파일을 저장할 때 설정된 외부 경로를 사용하도록 업데이트했습니다.

## 3. 기술적 설명 (비유로 설명)
기존 방식은 마치 **"새 책이 서점에 들어왔는데, 서점 주인이 내일 아침에 정리를 끝내야만 손님이 볼 수 있는 상황"**과 같았습니다. Spring Boot의 클래스패스는 실행 중에 새로 추가된 파일을 즉시 인식하지 못하는 특성이 있기 때문입니다.

이번 수정을 통해 **"서점 외부에 별도의 전시대(외부 디렉토리)를 마련하고, 손님이 언제든 바로 볼 수 있게 통로(리소스 핸들러)를 연결한 것"**과 같습니다. 이제 파일이 생성되는 즉시 브라우저에서 접근이 가능합니다.

## 4. 검증 결과
- `GalleryService`: `./upload/` 경로에 파일이 정상적으로 저장되고, `/upload/파일명`으로 서빙됨을 확인.
- `NoticeService`: 동일하게 외부 경로를 사용하여 즉시 반영 문제 해결.
- `WebMvcConfig`: 외부 경로와 기존 `static` 경로 모두 지원하여 하이브리드 리소스 서빙 구조 확보.
