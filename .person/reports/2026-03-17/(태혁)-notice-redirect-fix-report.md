# [공지사항] 게시물 등록 후 리다이렉트 오류 수정 보고서

## 1. 개요
공지사항 관리자 페이지에서 '게시하기' 버튼 클릭 시, 데이터는 저장되지만 리스트 화면으로 리다이렉트되지 않던 문제를 해결하였습니다.

## 2. 주요 변경 사항
- **AdminNoticeController.java**: `save` 메소드의 반환 타입을 `String`으로 설정하고 `redirect:/admin/notices`를 사용하여 PRG 패턴(POST-Redirect-GET)을 준수하였습니다.
- **save-form.mustache**:
    - 버튼(`id="btnNoticeSave"`) 클릭 이벤트 핸들러 추가.
    - Summernote 에디터의 HTML 내용을 `textarea`의 값으로 명시적 동기화.
    - 폼 데이터 유효성 검사 후 `$('#noticeSaveForm').submit()`을 통해 동기식으로 서버에 제출하여 리다이렉트 응답을 처리할 수 있도록 개선.

## 3. 검증 결과
- 공지사항 등록 시 H2 데이터베이스에 데이터가 정상적으로 저장됨을 확인.
- 등록 완료 후 `/admin/notices` (공지사항 목록) 화면으로 자동 이동됨을 확인.
- 리스트 화면에서 새로 등록된 공지사항이 최상단에 노출되는 것을 확인.

## 4. 비유로 설명하기
이 오류는 마치 **우체국에서 편지를 부칠 때, 편지는 통에 잘 넣었는데 정작 본인은 집에 가지 못하고 우체국 앞에 계속 서 있는 상황**과 같았습니다. 

이번 수정을 통해, 편지(공지사항 데이터)가 접수되면 우체국 직원(서버)이 "잘 접수되었습니다!"라고 말하며 고객(브라우저)에게 "이제 집(리스트 페이지)으로 돌아가셔도 좋습니다"라고 안내하도록 시스템을 고쳤습니다.
