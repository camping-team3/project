## 📝 작업 요약
관리자가 수많은 예약 데이터 속에서 원하는 내역을 빠르게 찾을 수 있도록 예약자명, 예약 ID, 체크인 날짜, 예약 상태를 기준으로 한 통합 검색 및 필터 기능을 구현했습니다.

## 🛠️ 주요 변경 사항
### 1. 동적 검색 파라미터 처리 (DTO & Controller)
- **AdminRequest.ReservationSearchDTO**: 검색 키워드, 날짜, 상태 값을 서버로 전달하기 위한 전용 객체를 도입했습니다.
- **상태 유지**: 검색 후에도 입력했던 검색어와 선택한 필터값이 화면에 그대로 유지되도록 모델 바인딩을 강화했습니다.

### 2. 효율적인 DB 필터링 (Repository)
- **복합 쿼리**: JPQL의 `NULL` 체크 로직을 활용하여, 사용자가 입력한 필터 조건이 있을 때만 쿼리에 반영되는 효율적인 검색 로직을 구현했습니다.
- **ID/이름 통합 검색**: 키워드 하나로 예약자의 '이름'과 '예약 ID'를 동시에 검색할 수 있어 사용성을 높였습니다.

### 3. 사용자 중심의 필터 UI (View & JS)
- **폼 연동**: 기존 디자인의 필터 영역을 실제 작동하는 `<form>`으로 전환했습니다.
- **자동 값 설정**: 페이지 로드 시 이전에 선택했던 예약 상태가 드롭다운에 자동으로 선택되어 있도록 JS 처리를 추가했습니다.

## 💡 비유로 설명하기
**"수천 권의 장부에서 포스트잇 찾기"**
예전에는 관리자가 모든 예약 내역이 적힌 두꺼운 장부를 일일이 넘겨가며 손님을 찾아야 했다면, 이제는 장부 맨 위에 '찾기 버튼'을 만든 것과 같습니다. "홍길동 손님 찾아줘"라고 하면 홍길동이 적힌 페이지들이 바로 펼쳐지고, "오늘 오는 손님만 보여줘"라고 하면 오늘 날짜가 적힌 장부만 따로 모아서 보여주는 식입니다. 덕분에 관리자는 더 이상 눈이 아프게 장부를 뒤질 필요가 없어졌습니다.
