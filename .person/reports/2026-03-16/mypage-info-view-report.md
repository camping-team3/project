# 📄 마이페이지 기본 정보 조회 기능 구현 보고서

- **날짜**: 2026-03-16
- **작성자**: handikim
- **작업 단계**: Phase 2 - User 도메인 마이페이지 정보 연동

## 1. 작업 요약
로그인한 사용자가 자신의 정보를 확인할 수 있도록 마이페이지 홈 화면에 실제 회원 데이터를 연동하였습니다.

## 2. 변경 사항
- **컨트롤러 보완**: `UserController.home()`에서 세션의 `sessionUser.id`를 사용하여 DB에서 회원 상세 정보를 조회하고, 이를 `user` 객체로 모델에 담아 뷰에 전달하도록 구현하였습니다.
- **템플릿 연동**: `mypage/home.mustache`에 하드코딩되어 있던 이름(김철수)과 역할을 `{{user.name}}`, `{{user.role}}`로 교체하여 로그인한 사용자의 정보가 동적으로 출력되게 하였습니다.

## 3. 검증 결과
- 로그인 후 마이페이지(/mypage) 진입 시 가입할 때 입력한 이름이 정상적으로 표시됩니다.
- 비로그인 상태에서 마이페이지 접근 시 로그인 페이지로 리다이렉트되는 방어 로직을 확인하였습니다.

## 4. 비유로 설명하기
마이페이지 정보 연동은 **"캠핑장 개인 사물함에 명찰을 다는 것"**과 같습니다.
이전에는 사물함에 다른 사람의 이름(김철수)이 붙어 있거나 비어 있었지만, 이제는 고객님이 체크인할 때 받은 손목 밴드(세션)를 대면 자동으로 고객님의 이름이 적힌 명찰이 나타나고 고객님만의 물건들이 보이도록 설정한 것입니다.
