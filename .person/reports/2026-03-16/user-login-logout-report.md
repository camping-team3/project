# 📄 로그인 및 로그아웃 기능 구현 보고서

- **날짜**: 2026-03-16
- **작성자**: handikim
- **작업 단계**: Phase 2 - User 도메인 로그인/로그아웃 및 세션 관리 구현

## 1. 작업 요약
가입된 회원이 서비스를 이용할 수 있도록 로그인 및 로그아웃 기능을 구현하였습니다. `HttpSession`을 통해 인증 상태를 유지하며, 권한(Role)에 따라 UI를 다르게 보여주는 처리를 포함하였습니다.

## 2. 변경 사항
- **응답 DTO 정의**: `UserResponse.LoginDTO`를 신설하여 세션에 저장할 최소한의 정보(ID, 이름, 권한)를 담았습니다. 또한 `DetailDTO`를 통해 마이페이지 등에 필요한 상세 정보를 제공합니다.
- **서비스 구현**: `UserService.login()`에서 아이디 존재 여부 확인 및 `BCrypt` 비밀번호 일치 검증 로직을 완성하였습니다.
- **컨트롤러 구현**:
    - `POST /login`: 로그인 성공 시 `sessionUser`라는 이름으로 세션 정보를 저장합니다.
    - `GET /logout`: 세션을 무효화(`invalidate`)하고 메인 페이지로 보냅니다.
    - `GET /mypage`: 세션 유무를 확인하고 상세 정보를 조회하여 뷰에 전달합니다.
- **UI 연동**:
    - `auth/login-form.mustache`: 폼 속성을 서버와 맞추고 아이디/비밀번호 입력 필드를 연결하였습니다.
    - `layout/header.mustache`: 세션 유무에 따라 '로그인' 버튼 또는 '마이페이지/로그아웃' 버튼이 보이도록 동적 렌더링을 적용하였습니다.

## 3. 검증 결과
- 비밀번호가 틀리거나 없는 아이디로 로그인 시도 시 적절한 예외 메시지가 발생합니다.
- 로그인 성공 시 헤더가 즉시 변경되며 마이페이지 접근이 가능해집니다.
- 로그아웃 클릭 시 세션이 소멸되어 다시 비로그인 상태로 전환됩니다.

## 4. 비유로 설명하기
로그인과 로그아웃은 **"캠핑장 체크인 및 체크아웃"**과 같습니다.
1. **체크인(로그인)**: 관리실에 들러 발급받은 아이디와 비밀번호를 보여줍니다. 관리자는 고객님의 암호화된 기록(BCrypt)과 대조하여 본인이 맞는지 확인합니다.
2. **이용권(세션) 발급**: 본인 확인이 되면 캠핑장 내 모든 시설을 이용할 수 있는 **"손목 밴드(세션)"**를 채워드립니다. 이 밴드를 차고 있으면 식당(예약하기)이나 수영장(마이페이지)에 갈 때마다 매번 신분증을 보여줄 필요가 없습니다.
3. **권한 확인**: 밴드에는 '일반' 또는 '관리자' 등급이 적혀 있어, 등급에 맞는 구역만 출입할 수 있게 합니다.
4. **체크아웃(로그아웃)**: 즐거운 캠핑을 마치고 밴드를 반납(세션 무효화)하면, 더 이상 회원 전용 시설을 이용할 수 없게 되며 일반 방문객 상태로 돌아갑니다.
