# 공지사항(Notice) 초기 데이터 정합성 수정 보고서

## 1. 개요
최근 공지사항(Notice) 기능에 추가된 **상단 고정(isTop)** 필드로 인해, 서버 구동 시 초기 데이터를 주입하는 `data.sql`에서 무결성 제약 조건(NOT NULL) 위반 에러가 발생하여 서버가 실행되지 않는 문제를 해결하였습니다.

## 2. 주요 수정 내용
- **대상 파일**: `src/main/resources/db/data.sql`
- **수정 범위**: 7. 공지사항 (notice_tb) INSERT 문
- **상세 변경 사항**:
    - `INSERT INTO notice_tb` 문에 `is_top` 컬럼 추가
    - 각 공지사항 데이터 행에 `true`(상단 고정) 또는 `false`(일반 공지) 값 명시
    - 컬럼명 오타 수정 (`is-top` → `is_top`)

## 3. 검증 결과
- **빌드 및 실행**: `./gradlew bootRun` 시 `dataSourceScriptDatabaseInitializer` 에러 없이 정상 구동 완료.
- **포트 확인**: 8080 포트 활성화 확인 및 런타임 접속 성공.
- **데이터 확인**: H2 콘솔 또는 서비스 화면에서 `is_top` 값에 따른 정렬(상단 고정)이 정상적으로 이루어짐을 확인.

## 4. 비유로 설명하기 (집짓기에 비유)
우리가 살 집(애플리케이션)에 **"중요 알림판(Notice)"**을 새로 만들면서, 모든 알림에는 **"빨간색 핀(isTop)"**을 꽂을지 말지 반드시 결정해야 한다는 새로운 규칙을 정했습니다. 

그런데 이사 올 때 가져온 짐 목록(data.sql)에는 예전 규칙대로 핀에 대한 정보가 하나도 없었습니다. 그래서 집 관리자(Spring Boot/H2)가 "핀 정보가 없으면 알림판을 세울 수 없다!"며 이사를 거부했던 상황입니다.

오늘 작업은 그 짐 목록에 일일이 "이건 핀을 꽂으세요", "이건 꽂지 마세요"라고 표시를 남겨준 것입니다. 이제 관리자가 만족하여 알림판을 예쁘게 세워주었습니다.
