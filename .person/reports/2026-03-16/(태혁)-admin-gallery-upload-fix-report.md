# 관리자 갤러리 이미지 업로드 기능 복구 보고서

## 1. 개요
관리자 페이지의 새 갤러리 게시글 작성 화면(`admin/gallery/new.mustache`)에서 이미지를 추가하기 위한 업로드 영역이 정적인 UI로만 구성되어 있어, 클릭이나 드래그 앤 드롭 시 아무런 반응이 없던 문제를 해결하였습니다.

## 2. 주요 수정 내용
- **대상 파일**: `src/main/resources/templates/admin/gallery/new.mustache`
- **UI 구조 개선**:
    - 실제 파일 선택 기능을 담당할 숨겨진 `<input type="file" multiple>` 추가.
    - 전체 입력 섹션을 데이터 전송을 위한 `<form>`으로 래핑.
- **JavaScript 로직 구현**:
    - **클릭 이벤트**: 업로드 영역 클릭 시 파일 탐색기가 트리거되도록 연결.
    - **드래그 앤 드롭**: `dragover`, `dragleave`, `drop` 이벤트를 통해 외부 파일 드롭 시 즉시 추가되도록 구현.
    - **동적 프리뷰**: 선택된 파일을 읽어 화면 하단에 미리보기를 생성하고, `×` 버튼으로 개별 삭제가 가능하도록 구현.
    - **파일 동기화**: `DataTransfer` 객체를 활용하여 프리뷰 삭제 시 실제 서버로 전송될 파일 목록도 동기화.

## 3. 검증 결과
- **UI 반응**: 업로드 영역 클릭 시 정상적으로 파일 선택창이 나타남.
- **드롭 동작**: 사진 파일을 드래그하여 놓았을 때 즉시 업로드 대기 목록에 추가됨.
- **프리뷰**: 이미지 미리보기가 생성되며, 삭제 시 목록에서 정상적으로 제외됨을 확인.
- **애니메이션**: `animate.css`를 통한 부드러운 UI 효과 확인.

## 4. 비유로 설명하기 (요리 재료 준비에 비유)
예전의 갤러리 작성 페이지는 예쁜 **"식재료 바구니(UI)"**만 그려져 있는 포스터 같았습니다. 아무리 재료(사진)를 담으려고 해도 그냥 종이 위에 올려놓는 것뿐이었죠.

이번 작업은 그 포스터 뒤에 진짜 **"바구니(파일 입력창)"**를 가져다 놓고, 재료를 던져 넣으면(드래그) 쏙 들어가게 통로를 만든 것입니다. 또한, 바구니에 담긴 재료들을 미리 확인해보고 마음에 안 드는 건 다시 뺄 수 있는 **"투명한 덮개(미리보기)"**도 달아준 셈입니다. 이제 요리사(관리자)는 재료를 마음껏 준비할 수 있게 되었습니다.
