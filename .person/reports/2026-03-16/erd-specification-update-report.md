# 📊 ERD 설계 명세 업데이트 보고서 (Documentation Sync)

시스템 고도화 과정에서 변경된 데이터베이스 스키마(`data.sql`)와 실제 엔티티 구조를 설계 문서인 `.person/docs/ERD.md`에 반영하여 동기화 작업을 완료했습니다.

## 2. 주요 변경 사항

### 🏗️ 공간 및 운영 관리 (Zone, Site, Season)
- **성수기(Season) 테이블 추가**: 성수기 기간을 별도로 관리하기 위한 테이블 명세를 추가했습니다.
- **구역(Zone) 속성 확장**: `base_people`(기준 인원), `extra_person_fee`(추가 요금) 필드를 명세에 포함했습니다.
- **사이트(Site) 상태값 추가**: `is_available`(예약 가능 여부) 속성을 추가하여 운영 상태 관리를 명확히 했습니다.

### 📅 예약 및 이미지 관리 (Reservation, Image)
- **예약(Reservation) 필드 고도화**: `people_count`(예약 인원), `visitor_name`(방문자명), `visitor_phone`(방문자 연락처) 속성을 추가하고, `COMPLETED`(이용 완료) 상태를 명세에 반영했습니다.
- **이미지(Image) 연관 관계 확장**: 기존 갤러리/리뷰 외에도 구역(`zone_id`)과 사이트(`site_id`)에 이미지를 연결할 수 있도록 외래키 관계를 추가했습니다.

## 3. 검증 결과
- **정확성**: 실제 Java 엔티티 클래스 및 `src/main/resources/db/data.sql`의 최신 컬럼 구조와 100% 일치함을 확인했습니다.
- **무결성**: 엔티티 관계도(Relationships) 섹션에 새롭게 추가된 1:N 관계(Zone-Image, Site-Image 등)를 모두 반영했습니다.
