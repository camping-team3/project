## 📝 작업 요약
`실시간 예약(new.mustache)` 페이지에서 사용자 경험을 개선하기 위해 사이트 선택 방식을 '다중 선택'에서 '단일 선택 및 토글(취소)' 방식으로 변경하고, 새로운 인원 추가 요금 정책을 JS 로직에 반영했습니다.

## 🛠️ 주요 변경 사항
### 1. 사이트 선택 메커니즘 변경 (JS)
- **단일 선택(Single Select)**: 쇼핑카트에 여러 물건을 담는 방식에서, '하나의 방'만 선택하는 호텔 예약 방식으로 변경했습니다. 다른 사이트를 클릭하면 기존 선택은 자동으로 내려놓습니다.
- **토글 취소(Toggle/Deselect)**: '선택됨' 버튼을 한 번 더 누르면 선택이 취소되도록 `deselectSite` 함수를 추가했습니다.

### 2. 스마트 가격 계산기 적용
- **공식**: `(기본 1박 요금 + (추가 인원 * 10만원)) * 숙박일수`
- **로직**: `Math.max(0, 선택인원 - 기준인원)`를 사용하여, 인원이 기준 이내일 때는 추가 요금이 0원이 되고, 기준을 초과할 때만 인당 10만원이 정확히 합산되도록 설계했습니다.

### 3. UI/UX 피드백 강화
- **버튼 상태**: 선택 시 '선택하기(투명)' → '선택됨(파란색)'으로 색상을 명확히 구분하여 사용자가 무엇을 선택했는지 직관적으로 알 수 있게 했습니다.
- **폼 전송**: 서버로 데이터를 보낼 때 `siteId`라는 이름으로 하나의 ID값만 안전하게 전달하도록 수정했습니다.

## 💡 비유로 설명하기
**"호텔 방 고르기"**
예전에는 뷔페 접시에 여러 음식을 담는 식이었다면, 이제는 호텔 체크인 카운터에서 딱 하나의 방 키를 고르는 방식입니다. 이미 키를 하나 들고 있는데 다른 방 키를 집으면 이전 키는 카운터에 반납하는 것과 같죠. 또한, 방의 기본 정원보다 사람이 더 오면 '추가 침대(Extra Bed)' 비용을 받는 것처럼, 초과 인원에 대해서만 정확히 추가 요금을 매기도록 계산기를 똑똑하게 고쳤습니다.
