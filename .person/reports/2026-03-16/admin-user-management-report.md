# 📄 관리자 회원 관리 및 권한 제어 기능 구현 보고서

- **날짜**: 2026-03-16
- **작성자**: handikim
- **작업 단계**: Phase 2 - User 도메인 관리자 기능(목록, 권한 변경)

## 1. 작업 요약
관리자가 전체 회원 목록을 확인하고, 특정 회원의 권한(일반 사용자 <-> 관리자)을 동적으로 변경할 수 있는 기능을 구현하였습니다.

## 2. 변경 사항
- **엔티티 확장**: `User.java`에 `updateRole` 비즈니스 메서드를 추가하여 권한 변경이 가능하도록 하였습니다.
- **서비스 구현**:
    - `findAll(Pageable)`: Spring Data JPA의 페이징 기능을 활용하여 효율적인 회원 목록 조회를 구현하였습니다.
    - `updateRole(id, role)`: 특정 회원의 등급을 변경하는 트랜잭션 로직을 추가하였습니다.
- **관리자 전용 컨트롤러**: `AdminUserController.java`를 신설하여 `/admin/users/**` 경로를 담당하게 하였습니다.
- **UI/UX 연동**:
    - `admin/user/list.mustache`: 실제 회원 데이터를 테이블에 바인딩하고 페이징 네비게이션을 연동하였습니다.
    - `admin/user/detail.mustache`: 회원 상세 정보를 출력하고, 권한 등급을 선택하여 즉시 변경할 수 있는 폼을 추가하였습니다.

## 3. 검증 결과
- 관리자 페이지의 사용자 목록에서 등록된 회원들이 정상적으로 출력됩니다.
- 회원 상세 페이지에서 권한을 'ADMIN'으로 변경 시 DB와 세션(재로그인 시)에 정상 반영됩니다.
- 페이징 처리를 통해 대규모 회원 데이터 환경에서도 성능을 유지할 수 있도록 설계하였습니다.

## 4. 비유로 설명하기
관리자 회원 관리는 **"캠핑장 전체 명부 관리 및 VIP 등급 부여"**와 같습니다.
1. **회원 명부(목록)**: 캠핑장에 머무는 모든 손님의 정보를 한눈에 볼 수 있는 커다란 게시판을 만들었습니다. 손님이 너무 많으면 한 페이지에 다 적을 수 없으니 여러 장(페이징)으로 나누어 정리했습니다.
2. **등급 변경(권한)**: 특정 손님에게 캠핑장 모든 시설을 관리할 수 있는 **"마스터 키(ADMIN)"**를 줄지, 일반 이용권만 줄지 결정하는 기능입니다. 관리자가 명부에서 손님을 골라 등급을 수정하면 그 즉시 해당 손님의 권한이 바뀝니다.
3. **상세 정보**: 손님의 연락처나 상태를 확인하여 캠핑장 운영에 필요한 조치를 취할 수 있는 상세 장부를 마련한 것입니다.
