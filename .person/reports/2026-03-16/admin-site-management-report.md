## 작업 요약
관리자 전용 구역 및 사이트 통합 관리 기능을 구현하였습니다. (메뉴 연동 제외)

## 변경 사항

### 1. Backend (Java)
- **`ZoneRepository`**: 구역과 소속 사이트를 한 번에 가져오는 Fetch Join 쿼리(`findAllWithSites`)를 포함하여 생성했습니다.
- **`SiteResponse`**: 관리자 화면의 계층 구조(구역 > 사이트)를 표현하기 위한 `AdminZoneDTO`와 `AdminSiteDTO`를 추가했습니다.
- **`SiteService`**: 모든 구역과 사이트 정보를 가공하여 반환하는 `findAllForAdmin()` 메서드를 구현했습니다.
- **`AdminController`**: `/admin/sites` 경로를 추가하여 관리자용 통합 관리 페이지를 매핑했습니다.

### 2. Frontend (Mustache)
- **`admin/site/list.mustache`**: 
  - 구역별 카드 섹션을 통해 구역명과 요금(평시/성수기) 정보를 한눈에 볼 수 있도록 구성했습니다.
  - 각 구역 하단에 소속된 사이트 목록을 테이블 형태로 출력하여 ID, 사이트명, 최대 인원을 확인할 수 있게 했습니다.
  - 구역 추가, 요금 수정, 사이트 추가/수정/삭제 등 향후 기능을 위한 UI 버튼들을 배치했습니다.

## 검증 결과
- `/admin/sites` 접속 시 DB에 등록된 구역(A, B, C)과 각 구역에 속한 6개의 사이트가 계층적으로 올바르게 표시되는 것을 확인했습니다.
- N+1 문제 없이 한 번의 쿼리로 모든 구역과 사이트 정보가 로드됨을 확인했습니다.

## 비유로 설명하는 현재 상태
이전까지는 관리자가 **"창고에 어떤 물건(사이트)이 있는지 확인하려면 직접 장부를 하나씩 찾아봐야 하는 상태"**였다면, 이제는 **"창고의 모든 구역별로 어떤 물건이 어디에 있고 가격이 얼마인지 한눈에 보여주는 모니터링 대시보드"**를 설치한 것과 같습니다. (단, 대시보드로 가는 버튼은 아직 달지 않은 상태입니다.)
