# 회원가입 아이디 중복 확인 기능 추가

- Date: 2026-03-16
- Reporter: handikim
- Task: Phase 2 User 도메인 - 회원가입 폼 중복 확인 기능 보완

## 작업 요약
회원가입 폼(`join-form.mustache`)에서 아이디(`username`)를 입력받는 폼 필드를 추가하고, 기존 폼 디자인을 해치지 않으면서 실시간으로 서버에 중복 여부를 질의할 수 있는 검증 기능을 추가했습니다.

## 변경 사항
1. **아이디 입력란 및 버튼 추가**: 
   - 이름과 이메일 주소 사이에 아이디(`username`) 입력란을 새롭게 추가.
   - Bootstrap의 `d-flex gap-2` 유틸리티를 활용해 입력란과 '중복 확인' 버튼을 기존 디자인 패턴(둥근 모서리, 옅은 테두리 등)에 맞게 가로로 나란히 배치.
2. **Form 속성 정상화**: 
   - 기존에 누락되었던 폼의 `action="/join" method="post"`와 내부 필드들의 `name` 속성(`name`, `username`, `email`, `phone`, `password`)을 모두 채워 넣음.
3. **AJAX 비동기 검증 (JavaScript)**:
   - `fetch`를 사용해 기존에 구현된 `GET /api/users/check-username` 엔드포인트를 호출.
   - 응답 상태(`response.ok`)에 따라 성공 시 초록색, 실패 시 빨간색 안내 메시지를 입력창 하단에 동적으로 출력.
4. **폼 제출 방어 로직 (Validation)**:
   - 중복 확인 버튼을 누르지 않거나 중복된 아이디일 경우(`isUsernameChecked === 'false'`), 폼이 제출(`submit`)되지 않도록 막고 경고창(`alert`)을 띄움.
   - 추가로 비밀번호와 비밀번호 확인이 일치하는지도 검사하는 로직 포함.

## 비유로 보는 작업 내용
마치 건물의 '출입증 발급 신청서'에 '희망하는 사원 번호' 적는 칸이 빠져 있어서 추가한 것과 같습니다. 신청서를 제출하기 전에 해당 사원 번호가 이미 다른 사람에게 발급되었는지 확인해주는 '버튼'과 '알림 전광판'을 신청서 옆에 예쁘게 달아주어, 접수 데스크(서버)가 헛걸음을 치지 않도록 보완했습니다.

## 검증 결과
- 중복 확인 버튼 클릭 시 API가 정상적으로 호출되며, 사용 가능/불가능 메시지가 UI에 즉시 반영됩니다.
- 아이디를 다시 수정하면 중복 확인 상태가 초기화(`false`)됩니다.
- 중복 확인을 마치지 않은 상태에서는 회원가입 완료 버튼을 눌러도 폼이 제출되지 않고 막힙니다.
