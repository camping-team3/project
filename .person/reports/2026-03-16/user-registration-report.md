# 📄 회원가입 기능 구현 보고서

- **날짜**: 2026-03-16
- **작성자**: handikim
- **작업 단계**: Phase 2 - User 도메인 회원가입 로직 구현

## 1. 작업 요약
사용자가 서비스를 이용하기 위해 필수적인 회원가입 기능을 구현하였습니다. 보안을 위해 `Spring Security`를 도입하여 비밀번호를 암호화하여 저장하며, 중복 아이디 가입을 방지하는 로직을 포함하였습니다.

## 2. 변경 사항
- **의존성 추가**: `build.gradle`에 `spring-boot-starter-security`를 추가하였습니다.
- **보안 설정**: `global/config/SecurityConfig.java`를 신설하여 `BCryptPasswordEncoder`를 빈으로 등록하고, 기본 인증 방식을 허용하도록 설정하였습니다. (인증은 추후 인터셉터에서 처리)
- **DTO 정의**: `UserRequest.JoinDTO`를 정의하여 회원가입에 필요한 필드(아이디, 이름, 이메일, 전화번호, 비밀번호)를 매핑하였습니다.
- **서비스 구현**: `UserService.join()`에서 아이디 중복 체크 및 비밀번호 암호화 후 DB에 저장하는 로직을 완성하였습니다.
- **컨트롤러 구현**: `UserController`에 회원가입 폼 이동 및 가입 처리(`POST /join`)를 추가하고 PRG 패턴을 적용하였습니다.
- **UI 보완**: `auth/join-form.mustache`의 폼 속성을 서버 로직과 동기화하고, 누락되었던 아이디 입력 필드를 추가하였습니다.

## 3. 검증 결과
- `BCryptPasswordEncoder`를 통해 비밀번호가 해시값으로 안전하게 저장됨을 설계하였습니다.
- 중복된 `username`으로 가입 시도 시 `RuntimeException`이 발생하여 비정상 가입을 차단합니다.
- 가입 완료 후 로그인 페이지로 정상적으로 리다이렉트됩니다.

## 4. 비유로 설명하기
회원가입은 **"캠핑장의 출입 카드를 발급받는 과정"**과 같습니다.
1. **아이디/비밀번호 입력**: 출입 카드에 사용할 아이디를 정하고 비밀번호(금고 번호)를 설정합니다.
2. **암호화**: 저희 캠핑장은 고객님의 비밀번호를 그대로 보관하지 않고, **특수 암호 기계(BCrypt)**를 이용해 누구도 알아볼 수 없는 복잡한 코드로 변환해 금고에 보관합니다. 설령 금고가 털리더라도 원래 비밀번호는 알 수 없게 말이죠!
3. **중복 체크**: 이미 같은 이름을 사용하는 캠핑객이 있는지 확인하여 혼선을 방지합니다.
4. **발급 완료**: 모든 절차가 끝나면 고객님은 이제 정식 회원으로서 캠핑장 문(로그인)을 두드릴 수 있게 됩니다.
