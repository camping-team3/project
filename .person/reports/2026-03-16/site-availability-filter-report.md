## 작업 요약
실시간 예약 페이지(`/reservations/new`)에서 사용자가 선택한 날짜에 맞춰 예약 가능한 사이트만 필터링하여 보여주는 기능을 구현하였습니다.

## 변경 사항

### 1. Backend (Site Domain 중심)
- **`SiteRepository`**: `Reservation` 엔티티를 서브쿼리로 참조하여, 특정 기간에 겹치는 예약이 **없는** 사이트만 조회하는 `findAvailableSites` 메서드를 추가했습니다. (JPQL `NOT IN` 사용)
- **`SiteService`**: 특정 기간의 예약 가능 사이트 목록을 조회하는 비즈니스 로직을 추가했습니다. (상태가 `PENDING`, `CONFIRMED`, `CANCEL_REQ`인 예약 건을 필터링 대상으로 지정)
- **`ReservationController`**: `checkIn`, `checkOut` 파라미터를 받아 `SiteService`를 호출하고, 가용 사이트 목록을 뷰에 전달하도록 수정했습니다.

### 2. Frontend (Mustache)
- **`reservation/new.mustache`**: 
  - 검색바: 기존 하드코딩된 버튼을 `<form>`과 `<input type="date">`로 변경하여 실제 날짜 선택이 가능하도록 했습니다.
  - 사이트 목록: `{{#availableSites}}` 루프를 사용하여 가용 사이트만 동적으로 출력하며, 검색 결과가 없을 경우 예외 메시지를 보여줍니다.
  - 요약 바: 선택한 체크인/체크아웃 날짜가 실시간으로 반영되도록 업데이트했습니다.

## 검증 결과
- `2026-03-01` ~ `2026-03-03` 기간 검색 시, `data.sql`에 등록된 기존 예약(A-1 사이트)이 있는 경우 해당 사이트가 목록에서 제외되는 것을 확인했습니다.
- 날짜 파라미터 없이 접근 시 오늘 날짜 기준으로 자동 검색됨을 확인했습니다.

## 비유로 설명하는 현재 상태
이전의 예약 페이지가 **"모든 방 번호가 적혀 있지만 직접 전화를 해서 빈 방인지 물어봐야 하는 종이 명부"**였다면, 이제는 **"날짜를 입력하면 현재 비어 있는 방만 불이 들어오는 자동 안내 시스템"**이 구축된 것입니다. 사용자는 더 이상 헛걸음하지 않고 예약 가능한 자리를 바로 찾을 수 있습니다.
