# 테스트 데이터 생성 보고서 (Test Data Generation Report)

## 1. 개요
- **작업 일시**: 2026-03-11
- **목적**: `ERD.md` 및 `PRD.md` 설계 명세를 기반으로 한 시스템 전 도메인의 테스트용 더미 데이터(testData.sql) 구축.
- **대상 파일**: `src/main/resources/db/testData.sql`

## 2. 주요 생성 데이터
| 도메인 | 주요 내용 | 비즈니스 케이스 |
| :--- | :--- | :--- |
| **회원 (User)** | 관리자(1), 일반(5), 탈퇴(1) | 권한 분리 및 익명화 정책 테스트 가능 |
| **공간 (Zone/Site)** | A(오토), B(글램핑), C(카라반) | 구역별 요금 및 사이트 인원 제한 테스트 |
| **예약 (Reservation)** | 총 5건 (과거~미래) | 이용완료, 이용중, 예약확정, 취소요청, 취소완료 상태 포함 |
| **결제/환불** | 결제 성공 4건, 환불 1건 | 포트원 연동 데이터 및 위약금 환불 이력 |
| **콘텐츠** | 공지(3), 갤러리(2), Q&A(2), 리뷰(1) | 답변 완료/미완료 및 다중 이미지(Image) 연동 |

## 3. 핵심 가이드 (SQL 주석 포함 사항)
- **상태값 (Enum)**: `status`, `role` 등 각 컬럼에 사용 가능한 모든 문자열 값과 의미를 주석으로 기록.
- **테이블 역할**: 각 `INSERT` 구문 시작점에 해당 테이블의 비즈니스적 역할 기술.
- **정합성**: FK 제약 조건을 준수하여 `User` -> `Zone` -> `Site` -> `Reservation` 순으로 데이터 삽입.

## 4. 향후 활용 방안
- 애플리케이션 기동 시 `testData.sql`이 자동 실행되도록 설정하여 즉각적인 UI/UX 테스트 가능.
- 예약 상태 전이(`PENDING` -> `CONFIRMED`) 및 리뷰 작성 권한 로직 검증에 활용.
