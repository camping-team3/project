# Final Report: Site & Zone 도메인 개발 완료 보고 (Phase 2)

## 1. 작업 개요
캠핑장 예약 시스템의 핵심인 공간(Zone)과 사이트(Site) 정보를 관리하고, 사용자가 날짜별 가용성을 조회하여 실시간 요금을 확인할 수 있는 전 과정을 성공적으로 구축함.

## 2. 주요 성과 및 구현 기능
### 2.1 관리자 통합 CRUD (Admin Panel)
- **구역(Zone) 관리**: 구역 등록, 수정(이름/요금 정책), 삭제 기능 구현.
- **사이트(Site) 관리**: 구역별 사이트 배치, 상세 정보 수정 및 삭제 기능 구현.
- **실시간 토글**: 날짜 입력 없이 버튼 하나로 구역별 성수기(Peak) 모드를 ON/OFF 할 수 있는 스위치 로직 구축.

### 2.2 사용자 검색 엔진 (User Search)
- **가용성 필터링**: `Reservation` 도메인의 코드를 건드리지 않고 `SiteRepository` 내 서브쿼리를 통해 중복 예약되지 않은 사이트만 골라내는 정밀 필터링 로직 구현.
- **동적 요금 산출**: 선택한 입실/퇴실 기간과 구역의 성수기 활성화 여부를 실시간 결합하여 총 숙박 요금을 자동 계산.
- **검색 UI**: 입실/퇴실 날짜 파라미터를 유지하며 검색 결과를 렌더링하는 `index.mustache` UI 연동.

### 2.3 아키텍처 및 코딩 규칙 확립
- **DTO 엔티티 생성자 패턴**: 서비스 레이어의 복잡도를 낮추기 위해 DTO 내부에서 엔티티를 받아 데이터를 가공하는 생성자 패턴을 전면 도입하고 `common-rule.md`에 공식 규칙으로 등록.
- **컨트롤러 통합**: 사용자용 기능과 관리자용 기능을 `SiteController`로 통합하되, `/admin/**` 경로 보안을 철저히 분리하여 관리 효율성 증대.

## 3. 핵심 설계 결정 (Decision Log)
- **Surgical Update**: 타 도메인(`Reservation`)과의 충돌을 방지하기 위해 해당 도메인 폴더 내 파일을 일절 수정하지 않는 외부 참조 방식의 쿼리 설계.
- **Immutability**: DTO의 모든 필드를 `final`로 선언하여 데이터 전달 과정에서의 오염 방지.
- **UX 우선**: 사용자가 가격 변동 이유를 납득할 수 있도록 '🔥 성수기 요금 적용 중' 배지 노출 정책 수립.

## 4. 최종 검증 결과
- 모든 API 및 뷰 렌더링에 대해 `gradlew classes` 컴파일 통과.
- `testData.sql` 기반의 기초 데이터와 연동하여 관리자 설정 값이 사용자 화면에 즉각 반영됨을 확인.

## 5. 인계 사항 (Handover)
- **예약 도메인 연동**: `index.mustache`의 예약하기 버튼 클릭 시 `siteId`, `checkIn`, `checkOut` 파라미터가 `/reservation/save-form`으로 전달됨. 예약 담당자는 이를 받아 예약 신청 로직을 구현하면 됨.
- **요금 고도화**: 향후 날짜별 성수기/비수기 자동 전환 기능 필요 시 `Season` 엔티티를 추가하여 확장 가능하도록 `SiteService` 구조를 유연하게 설계함.
