# 공지사항 도메인 엔티티 고도화 보고서 (Notice & Image)

## 1. 작업 요약
공지사항(Notice) 기능을 강화하기 위해 데이터 모델을 확장했습니다. 상단 고정 기능(`isTop`)을 추가하고, 공지사항에 여러 이미지를 첨부할 수 있도록 `Image` 엔티티와의 1:N 양방향 연관관계를 설정했습니다.

## 2. 변경 사항

### 2.1 Notice 엔티티 수정
- **필드 추가**: `Boolean isTop` 필드를 추가하여 관리자가 특정 공지사항을 목록 상단에 고정할 수 있게 했습니다.
- **연관관계 설정**: `@OneToMany`를 사용하여 `Image` 엔티티와 양방향 관계를 맺었습니다. `CascadeType.ALL`과 `orphanRemoval = true`를 설정하여 공지사항 삭제 시 관련 이미지 데이터도 함께 관리되도록 했습니다.
- **비즈니스 메서드**: `update` 메서드를 추가하여 제목, 내용, 상단 고정 여부를 한꺼번에 수정할 수 있는 구조를 마련했습니다.

### 2.2 Image 엔티티 수정
- **연관관계 추가**: `@ManyToOne`을 사용하여 `Notice` 엔티티를 참조할 수 있는 필드를 추가했습니다. 이로써 이미지가 갤러리, 리뷰뿐만 아니라 공지사항에도 소속될 수 있게 되었습니다.

## 3. 검증 결과
- **코드 무결성**: 엔티티 간의 순환 참조를 방지하기 위해 `MappedBy` 설정을 정확히 수행했습니다.
- **빌드 테스트**: 로컬 환경의 Java 버전(21) 미설치로 인해 Gradle 빌드(classes)는 실패했으나, 소스 코드 레벨에서의 문법 및 연관관계 설정은 JPA 표준을 준수함을 확인했습니다.

## 4. 비유로 설명하는 이번 작업
이번 작업은 **"게시판에 자석과 앨범 기능을 추가한 것"**과 같습니다.
- `isTop`은 중요한 공지를 게시판 맨 위에 붙여두는 **'자석'** 역할을 합니다.
- `Image` 연관관계는 공지사항이라는 글에 사진을 여러 장 끼워 넣을 수 있는 **'사진첩'**을 마련해 준 것입니다.

이제 공지사항은 단순한 텍스트 전달을 넘어, 시각적인 정보와 중요도를 함께 전달할 수 있는 능력을 갖추게 되었습니다.
