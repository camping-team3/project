# 공지사항 DTO 클래스 생성 보고서 (NoticeRequest & NoticeResponse)

## 1. 작업 요약
Phase 2의 2단계 첫 번째 작업으로, 공지사항의 요청과 응답을 담당할 DTO(Data Transfer Object) 클래스들을 생성했습니다.

## 2. 변경 사항

### 2.1 NoticeRequest.java 생성
- **SaveDTO**: 새로운 공지사항을 등록할 때 사용합니다. `toEntity()` 메서드를 포함하여 엔티티 변환을 용이하게 했습니다.
- **UpdateDTO**: 기존 공지사항을 수정할 때 사용합니다.
- **설계 특징**: `@Getter`와 `@Builder`를 적용하여 객체 생성 시 가독성을 높이고 불변성을 유지하도록 설계했습니다.

### 2.2 NoticeResponse.java 생성
- **ListDTO**: 공지사항 목록 조회 시 필요한 핵심 필드(ID, 제목, 상단 고정 여부, 작성일)만 포함하여 응답 속도를 최적화했습니다.
- **DetailDTO**: 공지사항 상세 조회 시 내용과 첨부된 이미지 목록(`ImageDTO`)까지 모두 포함하도록 설계했습니다.
- **ImageDTO**: 공지사항에 첨부된 이미지 정보를 전달하기 위한 전용 DTO를 내부에 포함했습니다.

## 3. 검증 결과
- **컴파일 테스트**: `gradlew classes`를 통해 새로 생성된 DTO들이 기존 엔티티(`Notice`, `Image`)와 정상적으로 연동되며 컴파일 오류가 없음을 확인했습니다.
- **규칙 준수**: 프로젝트 공통 규칙인 `@Getter`, `@Builder` 사용을 준수했습니다.

## 4. 비유로 설명하는 이번 작업
이번 작업은 **"공지사항을 전달할 전용 서류 봉투와 양식을 만든 것"**과 같습니다.
- `NoticeRequest`는 관리자가 공지사항을 작성해서 보낼 때 사용하는 **'작성 양식'**입니다.
- `NoticeResponse`는 사용자가 공지사항을 읽을 때 보기 좋게 정리해서 담아주는 **'서류 봉투'**입니다. 특히 목록용 작은 봉투와 상세 내용용 큰 봉투를 따로 만들어 효율성을 높였습니다.

이로써 비즈니스 로직 구현을 위한 데이터 전달 체계가 마련되었습니다.
