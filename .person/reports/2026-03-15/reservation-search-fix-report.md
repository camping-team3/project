# Report: 예약 검색 기능 보완 및 NPE 해결
- **Date**: 2026-03-15
- **Reporter**: parkcoding
- **Status**: Task 1-3 보완 완료

## 📝 작업 요약
`/reservations/new` 페이지 접속 시 발생하던 `NullPointerException`을 해결하고, 검색 필터와 사이트 리스트 간의 데이터 바인딩을 완료했습니다.

## 🛠️ 변경 사항
1. **Controller 중심 기본값 설정**: `ReservationController`에서 `checkIn`, `checkOut`, `peopleCount`의 기본값을 직접 세팅하여 뷰와 서비스가 일관된 데이터를 공유하도록 수정했습니다.
2. **DTO 확장**: `SiteResponse.ListDTO`에 `pricePerNight`, `totalPrice` 필드를 추가하여 1박 요금과 총 숙박 요금을 화면에 표시할 수 있게 했습니다.
3. **디자인 보존 및 기능 연결**: `new.mustache`의 기존 HTML/CSS 디자인을 100% 유지하면서, `form` 태그를 이용한 GET 방식 검색 기능을 구현했습니다.
4. **사이드바 요약**: 현재 검색 중인 일정과 인원 정보가 예약 요약 사이드바에 실시간으로 반영되도록 개선했습니다.

## ✅ 검증 결과
- `/reservations/new` 최초 접속 시 오늘 날짜 기준으로 사이트 자동 조회 확인.
- 날짜 및 인원 변경 후 '검색하기' 클릭 시 필터 조건이 유지되며 사이트 리스트가 갱신되는지 확인.
- 각 사이트별 1박 요금 및 숙박 일수(nights)가 곱해진 총 요금이 정확히 계산됨을 확인.
