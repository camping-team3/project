# 예외 처리 및 대용량 데이터 안정화 보고서 (Stability Enhancement)

## 1. 작업 요약
Phase 2의 5단계 두 번째 작업으로, 공지사항 도메인의 기술적 안정성을 높이기 위해 데이터베이스 저장 용량을 확장하고 파일 업로드 관련 예외 처리 로직을 보완했습니다.

## 2. 변경 사항

### 2.1 대용량 텍스트 지원 (Notice 엔티티)
- **필드 타입 변경**: `content` 필드의 타입을 `TEXT`에서 `LONGTEXT`로 상향 조정했습니다. Summernote 에디터를 통해 이미지나 복잡한 서식이 포함된 긴 글을 작성하더라도 데이터가 잘리거나 오류가 발생하지 않도록 충분한 저장 공간을 확보했습니다.

### 2.2 전역 예외 처리 보완 (GlobalExceptionHandler)
- **파일 크기 초과 처리**: `MaxUploadSizeExceededException` 핸들러를 추가했습니다. 사용자가 설정된 서버 제한보다 큰 파일을 업로드할 경우, 서버 오류 페이지 대신 "업로드 가능한 파일 크기를 초과했습니다"라는 친절한 안내 메시지를 alert 창으로 띄우도록 개선했습니다.
- **응답 방식 유지**: 기존 프로젝트의 방식에 따라 일반 요청과 AJAX 요청을 구분하여 적절한 응답(HTML script 또는 JSON)을 반환하도록 설계했습니다.

## 3. 검증 결과
- **컴파일 테스트**: `gradlew classes` 명령을 통해 엔티티 속성 변경과 예외 핸들러 추가 후에도 시스템이 정상적으로 빌드됨을 확인했습니다.
- **안정성 확인**: 데이터베이스 계층과 전역 예외 처리 계층의 보완을 통해 실서비스 운영 시 발생할 수 있는 엣지 케이스들을 사전에 대비했습니다.

## 4. 비유로 설명하는 이번 작업
이번 작업은 **"안내판의 저장 창고를 늘리고, 무거운 짐을 든 방문객을 위한 안내 데스크를 만든 것"**과 같습니다.
- 예전에는 창고(TEXT)가 좁아서 긴 글을 보관하기 힘들었지만, 이제는 아주 큰 창고(LONGTEXT)로 개조하여 아무리 긴 공지사항도 안전하게 보관할 수 있습니다.
- 또한, 게시판에 너무 무거운 사진(대용량 파일)을 붙이려다 실패한 방문객에게 무작정 안 된다고 화내는 대신, 친절하게 "조금 더 가벼운 사진으로 준비해 주세요"라고 안내해 주는 서비스 데스크를 마련한 셈입니다.

이로써 공지사항 서비스의 기술적 안정화 작업이 마무리되었습니다.
