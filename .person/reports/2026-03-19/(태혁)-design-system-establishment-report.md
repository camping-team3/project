# [REPORT] 디자인 시스템 수립 및 UI/UX 개발 표준 구축

- **Date**: 2026-03-19
- **Reporter**: xogurrh012
- **Task**: 디자인 토큰, 룰(Rule), 레퍼런스(Reference) 체계 수립

---

## 📝 작업 요약
팀 프로젝트의 시각적 일관성을 유지하고 협업 효율성을 높이기 위해 **'디자인 토큰'**과 **'UI 개발 표준'**을 수립했습니다. 이제 모든 팀원과 AI는 정해진 규칙에 따라 UI를 구현하게 됩니다.

## 🛠️ 주요 변경 사항
### 1. 디자인 토큰(Design Tokens) 정의
- `static/css/common.css`의 CSS 변수를 활용하여 브랜드 컬러(`--fh-primary`), 둥글기(`--fh-radius`), 배경색 등을 표준화했습니다.
- 하드코딩된 색상(#)이나 수치(px) 대신 **변수명**을 사용하여 유지보수성을 극대화했습니다.

### 2. 디자인 룰(`design-rule.md`) 수립
- **필수 절차 도입**: UI 작업 전 레퍼런스 확인 및 사용자 확인 질문 과정을 의무화했습니다.
- **목업 사전 승인**: 실제 코드 반영 전 **'텍스트 박스 다이어그램'**과 **'전체 소스 코드'**를 제시하여 사용자 승인을 받는 단계를 추가했습니다.

### 3. 레퍼런스(`.ai/references/`) 체계 구축
- 자주 쓰이는 UI(예: 어드민 게시글 목록)의 표준 코드를 레퍼런스 파일로 만들어, 필요할 때마다 복사하여 사용할 수 있는 '샘플북'을 마련했습니다.

---

## 💡 쉬운 설명 (비유)
이번 작업은 우리 프로젝트라는 **'도시'**를 짓기 위한 **'건축법'**을 제정하고, **'표준 설계도'**를 만든 것과 같습니다.
- **디자인 토큰**: "모든 건물 외벽은 이 페인트(변수)만 써야 한다"는 약속입니다.
- **디자인 룰**: "건물을 짓기 전 반드시 건축주(사용자)에게 청사진(목업)을 보여주고 허락을 받아라"는 절차입니다.
- **레퍼런스**: "아파트를 지을 때는 이 모범 설계도(샘플 코드)를 참고하라"는 가이드북입니다.

이제 누가 건물을 지어도 도시 전체의 풍경(UI)이 조화롭게 유지될 것입니다. 🏕️
