# Phase 3 Reservation 도메인 확장 (1단계: Enum 상태값 확장 및 정의)

**Date**: 2026-03-19
**Reporter**: parkcoding
**Task**: 1-1. Enum 상태값 확장 및 정의

## 1. 작업 요약
예약 변경 프로세스를 지원하기 위해 기존 예약 상태(`ReservationStatus`)를 확장하고, 관리자 승인 단계를 관리할 새로운 요청 상태(`RequestStatus`)를 정의했습니다.

## 2. 변경 사항
- **`ReservationStatus` 수정**: `CHANGE_REQ`(변경 요청 중) 상태를 추가하여 관리자가 승인하기 전까지의 상태를 명확히 표현했습니다.
- **`RequestStatus` 생성**: `PENDING`, `APPROVED`, `REJECTED` 상태를 정의하여 변경/취소 요청의 처리 단계를 구조화했습니다.
- **UI 친화적 설계**: `RequestStatus`에 `description` 필드를 추가하여 화면에 한글로 상태를 쉽게 노출할 수 있도록 했습니다.

## 3. 검증 결과
- 빌드 오류 없이 Enum 타입이 정상적으로 정의되었음을 확인했습니다.
- 패키지 컨벤션(`domain/reservation/enums/`)을 준수하여 생성되었습니다.

## 4. 비유로 보는 작업 내용
마치 레스토랑에서 **'주문 완료'** 상태만 있다가, 손님이 **'메뉴 변경 요청'**을 했을 때 주방장이 승인하기 전까지의 **'메뉴 변경 대기'**라는 새로운 알림표를 만든 것과 같습니다. 이제 주방장은 이 알림표를 보고 '승인' 또는 '거절'을 명확히 표시할 수 있게 되었습니다.
