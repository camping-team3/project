# Reporter: Gemini CLI

## 작업 요약
관리자 QnA 상세 페이지(답변 페이지)에서 데이터가 출력되지 않던 버그를 수정하였습니다.

## 변경 사항
- **AdminController**: `qnaAnswer` 메서드에서 누락되었던 데이터 조회 및 모델 바인딩 로직(`qnaService.findById`)을 추가하였습니다.

## 검증 결과
- `admin/qna/{id}/answer` 경로 접속 시 `qna` 객체가 정상적으로 모델에 담겨 전달됨을 확인.
- `answer.mustache` 템플릿의 `{{#qna}}` 섹션이 정상적으로 렌더링되어 고객명, 제목, 내용 등이 출력됨을 구조적으로 확인.

## 비유를 통한 설명
마치 **'빈 서류 봉투(Empty Model)'**만 들고 회의실에 들어갔던 상황입니다. 템플릿(회의 참석자)은 봉투 안에 내용이 있을 것이라 기대했지만 아무것도 없어서 발표를 할 수 없었습니다. 이제 봉투 안에 **'고객 문의 상세 내용(Qna Detail)'**을 제대로 채워서 넣어주었습니다.
