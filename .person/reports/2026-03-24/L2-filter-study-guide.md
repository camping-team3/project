# [기술 학습 보고서] 비동기 AI 정밀 분석 (L2 Filter) 아키텍처 및 핵심 기술

**작성자:** 김현지
**날짜:** 2026-03-24  
**주제:** Spring Boot 비동기 처리와 LLM(Large Language Model) 연동 기술 분석

---

## 1. 개요 (Introduction)

'2단계: 비동기 AI 정밀 분석'은 사용자가 리뷰를 등록한 '직후'에 실행됩니다. 사용자는 기다리지 않고 즉시 다음 화면으로 넘어가지만, 서버 내부에서는 AI가 해당 글을 정밀하게 읽고 위험 점수를 매깁니다. 이 과정에는 **비동기 프로그래밍(Asynchronous Programming)**과 **프롬프트 엔지니어링(Prompt Engineering)** 기술이 핵심적으로 사용되었습니다.

---

## 2. 핵심 기술 스택 (Technology Stack)

### 2.1 Spring `@Async` (비동기 처리)
- **개념**: 특정 메서드를 별도의 스레드(Thread)에서 실행하게 만드는 Spring 프레임워크의 기능입니다.
- **도입 이유**: AI 분석(Gemini API 호출)은 네트워크 통신이 수반되어 약 1~2초가 소요됩니다. 이를 동기식으로 처리하면 사용자는 1~2초 동안 '모래시계'를 보며 기다려야 합니다. `@Async`를 통해 분석은 배경에서 돌리고 사용자는 즉시 응답을 받게 합니다.

### 2.2 Google Gemini 1.5 Flash (LLM API)
- **개념**: 구글의 최신 거대 언어 모델입니다.
- **도입 이유**: 'Flash' 모델은 추론 속도가 매우 빠르고 비용이 저렴하여 실시간에 가까운 분석 작업에 최적화되어 있습니다. 단순 단어 매칭이 아닌 '비꼬는 말투'나 '교묘한 비방' 등의 문맥을 이해합니다.

### 2.3 프롬프트 엔지니어링 (Prompt Engineering)
- **개념**: AI가 우리가 원하는 정확한 결과를 내놓도록 지시문을 정교하게 설계하는 기술입니다.
- **우리 코드의 사례**: "다음 리뷰의 공격성 수치를 1~5 사이의 숫자로만 응답해줘"와 같은 페르소나와 제약 조건을 부여하여 시스템이 바로 처리 가능한 데이터를 얻어냅니다.

---

## 3. 작동 프로세스 (Workflow)

1.  **Trigger**: 사용자가 리뷰 저장/수정을 완료합니다.
2.  **Hand-off**: 메인 스레드는 `analyzeReviewAsync` 메서드를 호출하고 즉시 사용자에게 "성공" 응답을 보냅니다.
3.  **Analysis**: 백그라운드 스레드에서 Gemini API에 리뷰 텍스트를 전달합니다.
4.  **Parsing**: AI의 답변(텍스트)에서 숫자만 추출하여 데이터화합니다.
5.  **Update**: 분석된 위험 점수(1~5점)를 DB의 `ai_danger_score` 컬럼에 업데이트합니다.

## 4. 학습을 위한 핵심 키워드 (Topics for Study)

이 기능을 완전히 이해하고 직접 구현하기 위해 다음 내용들을 공부해보시는 것을 추천합니다.

### ① 비동기 프로그래밍과 스레드 풀 (Thread Pool)
- 왜 무분별한 스레드 생성은 위험한가?
- Spring의 `ThreadPoolTaskExecutor`를 설정하여 비동기 작업의 최대 개수를 제한하는 방법.

### ② REST API 통신 (`RestTemplate` vs `WebClient`)
- 자바에서 외부 서버(AI 서버)와 HTTP 통신을 주고받는 규격과 방법.
- JSON 데이터를 자바 객체로 변환(Parsing)하는 방법 (Jackson Library 등).

### ③ 데이터베이스 격리 수준과 동시성 제어
- 비동기 스레드에서 데이터를 수정할 때, 메인 스레드와 데이터 충돌이 일어나지 않게 관리하는 방법. (`@Transactional`의 전파 속성)

### ④ API 보안 및 환경 변수 관리
- API Key와 같은 민감한 정보를 소스 코드에 직접 적지 않고 `${GEMINI_API_KEY}`와 같이 환경 변수로 관리하는 보안 습관.

## 5. 결론 및 조언

현재 구현된 L2 Filter는 **'보이지 않는 곳에서 똑똑하게 일하는 서버'**를 만드는 좋은 예시입니다. 특히 AI 서버가 응답이 없거나 에러가 나더라도 사용자 서비스에는 지장이 없도록 설계된 **'결함 허용(Fault Tolerance)'** 관점을 중점적으로 살펴보시면 실력 향상에 큰 도움이 될 것입니다.
