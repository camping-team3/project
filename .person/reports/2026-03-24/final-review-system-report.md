## 1. 구현 기능 요약 (What we built)

### ① 마이페이지 리뷰 관리 최적화
- **정상화**: 리뷰 목록에서 사용자 프로필(이름, 등급)이 누락되어 렌더링 오류가 발생하던 문제를 해결했습니다.
- **UI 개선**: 중복된 더미 데이터를 제거하고, 실제 DB 데이터만 깔끔하게 노출되도록 레이아웃을 정리했습니다.
- **연동**: '예약 내역' 목록에서 이용 완료된 항목에 한해 [리뷰 작성하기] 버튼을 동적으로 노출하고, 작성 완료 시 상태를 변경하는 로직을 추가했습니다.

### ② 컴파운드(Compound) 필터링 시스템 🚀
단순 단어 매칭과 고도의 AI 분석을 결합한 2단계 방어 체계입니다.
- **L1 (실시간 차단)**: 등록/수정 버튼 클릭 즉시 욕설을 감지하여 저장을 원천 차단합니다.
- **L2 (AI 정밀 분석)**: 저장이 완료된 후, AI가 문맥을 읽고 위험 점수(1~5점)를 매겨 관리자에게 보고합니다.

### ③ 관리자 리뷰 대시보드 개편
- **토글 상세 보기**: 리스트 행을 클릭하면 리뷰 전문과 AI 분석 상세(게이지 바)가 나타나는 아코디언 UI를 구현했습니다.
- **위험도 시각화**: 점수에 따라 배지 색상을 변경(초록/빨강)하여 관리자가 즉시 조치할 수 있게 돕습니다.

## 2. 핵심 기술 용어 설명 (Glossary)

학습에 도움이 되도록 이번 작업에 쓰인 주요 용어들을 정리했습니다.

| 용어 | 설명 |
| :--- | :--- |
| **Compound Mode** | '복합 모드'라는 뜻으로, 여기서는 속도가 빠른 **단순 필터**와 정확도가 높은 **AI 분석**을 섞어서 사용하는 방식을 의미합니다. |
| **@Async (비동기)** | 메인 작업(리뷰 저장)이 끝난 뒤, 배경(Background)에서 별도로 돌아가는 방식을 말합니다. 사용자는 AI 분석을 기다릴 필요 없이 즉시 다음 화면으로 넘어갈 수 있습니다. |
| **L1/L2 Filter** | **L1(Layer 1)**은 입구에서 막는 차단막, **L2(Layer 2)**는 안에서 정밀하게 검사하는 보안 요원으로 비유할 수 있습니다. |
| **Regex (정규표현식)** | 텍스트에서 특정 패턴을 찾는 기술입니다. 이번에 "시!발", "개 2새끼"처럼 공백이나 특수문자를 섞은 우회를 잡기 위해 사용했습니다. |
| **Transaction (트랜잭션)** | "전부 성공하거나, 아니면 아예 안 한 것처럼 되돌리거나" 하는 단위입니다. 이번에 AI가 예전 글을 읽지 않도록 트랜잭션 종료 시점을 제어하는 것이 핵심이었습니다. |
| **Prompt Engineering** | AI가 우리가 원하는 대로 행동하도록 질문(지시문)을 설계하는 기술입니다. 이번에 "악성 리뷰 감별사" 페르소나를 부여해 검열 강도를 높였습니다. |

## 3. 문제 해결 과정 (How we fixed the "Stale Data" issue)

가장 어려웠던 부분은 **"수정했는데 왜 AI가 인식을 못 하는가?"**였습니다.

### [문제 현황]
1. `ReviewService.update()` (트랜잭션 안)에서 수정을 하고 곧바로 AI를 불렀습니다.
2. 하지만 DB는 트랜잭션이 완전히 끝나야(Commit) 데이터가 물리적으로 바뀝니다.
3. AI 스레드는 너무 빨라서, DB가 바뀌기도 전에 **수정 전의 깨끗한 옛날 글**을 읽어버렸습니다.

### [해결책: 2중 안전 장치]
1. **시점 변경**: AI 호출을 서비스(`Service`)가 아닌 **컨트롤러(`Controller`)**로 옮겼습니다. 컨트롤러는 서비스의 모든 작업(커밋)이 완벽히 끝난 뒤에 실행됩니다.
2. **확인 사살 (Confirmation Kill)**: AI에게 내용을 넘기기 직전에 `reviewService.findById()`를 호출해 **DB에서 갓 저장된 최신 데이터를 강제로 한 번 더 읽어와서** 전달하도록 설계했습니다.

## 4. 주요 수정 코드 요약

### `ReviewController.java` (AI 호출 컨트롤 타워)
```java
// 저장이 끝난 후 최신 데이터를 강제로 다시 읽어와 AI에게 전달
Review updatedReview = reviewService.update(id, user, req);
Review latestReview = reviewService.findById(updatedReview.getId()); // 최신본 재조회
aiAnalysisService.analyzeReviewAsync(latestReview.getId(), latestReview.getContent());
```

### `AiAnalysisService.java` (엄격한 검열 지침)
```java
// AI에게 주는 강력한 페르소나
String prompt = "너는 온라인 커뮤니티의 전문 검열관이야. ... " +
                "비속어가 직접적으로 없더라도, 문맥상 비꼬거나 불쾌감을 준다면 반드시 4~5점을 줘.";
```
