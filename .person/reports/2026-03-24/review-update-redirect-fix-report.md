# Review Update Redirect Fix Report

Reporter: Gemini-CLI
Date: 2026-03-24

## 작업 요약
- 리뷰 수정 시 발생하던 알 수 없는 로그인 페이지 리다이렉트 현상을 해결하고, 에러 발생 시 사용자에게 구체적인 메시지를 전달하도록 개선하였습니다.

## 변경 사항
- **ReviewController.java & ReviewService.java**:
    - 기존에 사용하던 자바 표준 예외인 `IllegalArgumentException`을 프로젝트 전용 예외인 `Exception400`으로 모두 교체하였습니다.
    - `Exception400`은 `GlobalExceptionHandler`에 의해 처리되며, 사용자에게 `alert` 창으로 에러 메시지를 보여주고 이전 페이지를 유지합니다.
    - 이를 통해 유효성 검사 실패(글자 수 부족 등)나 권한 오류 시 로그인 페이지로 튕기던 현상을 방지하였습니다.

## 검증 결과
- 이제 사용자가 리뷰 수정 폼에서 잘못된 값을 입력하거나 처리 중 오류가 발생할 경우, 로그인 페이지로 이동하는 대신 "내용은 10자 이상 작성해주세요"와 같은 구체적인 안내 메시지를 팝업으로 확인할 수 있습니다.
- 이는 코어(Global) 코드를 수정하지 않고 도메인 내부의 예외 타입만 변경하여 시스템의 안정성을 유지하면서 문제를 해결한 사례입니다.

---
비유로 설명하자면:
기존에는 경찰(에러 처리기)이 "모르는 가방(자바 표준 예외)"을 들고 있는 사람을 무조건 수상하게 여겨 경찰서(로그인 페이지)로 연행해갔습니다. 우리는 이 가방에 "신분증(Exception400)"을 넣어주어, 경찰이 신원을 확인하고 "무엇이 잘못되었는지" 친절하게 안내한 뒤 돌려보낼 수 있게 조치한 것입니다.
