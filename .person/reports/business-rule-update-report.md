# 비즈니스 규칙 최신화 보고서 (Business Rule Update Report)

## 1. 개요
- **작업 일시**: 2026-03-11
- **목적**: 독립 도메인 패키지 구조(`notice`, `gallery`, `qna`, `review`, `image`)에 맞춰 비즈니스 규칙을 재정의하고 최신 시스템 상태를 반영.
- **대상 파일**: `.ai/rules/business-rule.md`

## 2. 주요 업데이트 내용
| 도메인 | 주요 규칙 및 변경 사항 |
| :--- | :--- |
| **예약 라이프사이클** | `PENDING`, `CONFIRMED`, `CANCEL_REQ`, `CANCEL_COMP` 상태 전이 및 Redis 선점 락(5분) 명시. |
| **요금 및 환불** | 성수기 가격(`peak_price`) 적용 기준 및 위약금 비율(7일전 0%, 3~6일전 50%, 당일~2일전 100%) 정의. |
| **리뷰 (Review)** | 예약 상태가 `CONFIRMED`이며 이용일이 지난 경우에만 작성 가능하도록 제한. |
| **Q&A (Qna)** | 답변(`Comment`) 등록 시 `isAnswered` 자동 상태 전이 및 수정 제한 규칙 추가. |
| **이미지 (Image)** | UUID 기반 저장 원칙 및 갤러리/리뷰와의 1:N 관계 명시. |
| **회원 (User)** | `ANONYMOUS` 상태를 통한 데이터 이력 보존 및 개인정보 마스킹 정책 반영. |

## 3. 적용 효과
- 개발 시 참조할 수 있는 비즈니스 가이드라인이 현재 코드 구조와 완전히 일치됨.
- 예약 상태 전이 및 환불 승인 등 복잡한 비즈니스 로직의 기준점 마련.
- 다중 이미지 및 리뷰 작성 권한 등 누락되었던 핵심 규칙 보완.
