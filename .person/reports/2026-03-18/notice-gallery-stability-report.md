# 📄 공지사항 및 갤러리 도메인 안정화 보고서

## 1. 작업 요약
공지사항(Notice)과 갤러리(Gallery) 도메인의 안정성과 코드 품질을 높이기 위해 발견된 잠재적 오류를 수정하고 구조를 개선했습니다. 특히 물리적 파일 삭제 누락 문제와 데이터 정합성 검증을 강화하는 데 집중했습니다.

## 2. 세부 변경 사항

### 🛠️ 물리적 파일 삭제 기능 도입 (Bug Fix & Optimization)
- **`FileUtil.java`**: 파일 업로드 및 물리적 삭제 로직을 공통 유틸리티로 분리하여 중복 코드를 제거했습니다.
- **서비스 계층 적용**: 갤러리 및 공지사항의 `update` 시 이미지 교체(삭제)가 발생하거나, 게시글 전체 `delete` 시 서버 내 실제 파일(`upload/` 폴더)이 함께 삭제되도록 로직을 보완하여 스토리지 낭비를 방지했습니다.

### 🛡️ DTO 데이터 검증(Validation) 강화 (Stability)
- **의존성 추가**: `build.gradle`에 `spring-boot-starter-validation`을 추가했습니다.
- **DTO 제약 조건**: `SaveDTO`와 `UpdateDTO`의 제목, 내용, 촬영일 등 필수 필드에 `@NotBlank`, `@Size` 어노테이션을 적용했습니다.
- **이미지 필수 조건**: 특히 포토 갤러리(`Gallery`)의 경우, 작성 및 수정 시 최소 한 장 이상의 사진이 업로드되거나 남아있어야 하도록 백엔드(DTO/Service)와 프론트엔드(JS) 양측에서 검증을 강화했습니다.

### 🧩 엔티티 캡슐화 및 관계 설정 리팩토링 (Convention)
- **`Image.java`**: 무분별한 공용 Setter(`setGallery` 등)를 제거하고, 관계 설정을 위한 명확한 이름의 메서드(`updateGallery` 등)로 대체했습니다.
- **연관관계 편의 메서드**: `Gallery`와 `Notice` 엔티티에서 `Image`와의 관계를 맺을 때 변경된 명칭을 사용하도록 수정하여 객체의 안정성을 높였습니다.

## 3. 검증 결과
- **물리 파일 삭제**: 게시글 삭제 시 `upload/` 폴더 내 파일이 정상적으로 제거됨을 확인했습니다.
- **데이터 검증**: 빈 제목이나 내용 전송 시 `Exception400`이 발생하며 브라우저에 `alert` 창이 정상 노출됨을 확인했습니다.
- **사진 필수 체크**: 사진을 모두 삭제한 상태로 갤러리 수정을 시도할 경우 경고 메시지와 함께 제출이 차단됨을 확인했습니다.
