# plan 스킬 업데이트 리포트
2026-03-14
Reporter: parkcoding
작업 단계: plan/SKILL.md 규칙 강화 및 구조 개편

## 작업 요약
`plan` 스킬의 워크플로우를 '초기 계획 -> 세부 태스크 실행 -> 사후 처리'의 3단계로 구조화하고, 각 단계별 엄격한 준수 사항을 명시하여 개발 프로세스의 투명성과 안전성을 강화했습니다.

## 변경 사항
- **초기 계획 단계 (A)**: Task 분할 및 `TODO.md` 기록 절차 구체화.
- **세부 태스크 실행 단계 (B)**: 구현 전략 설명 시 코드 블록 금지 및 전체 소스 코드(Full Code) 제안 의무화.
- **사후 처리 단계 (C)**: 리포트 작성, 상태 동기화, 자동 커밋 순서 강제.
- **핵심 지침 추가**: 코드 품질(생략 금지), 맥락 독립성, 작업 재개 로직 및 수동적 할 일 선정 원칙 명시.

## 검증 결과
- `plan/SKILL.md` 파일이 의도한 대로 상세하게 수정되었음을 확인했습니다.
- 새로운 규칙에 따라 사후 처리(리포트 작성 및 커밋 제안)를 수행 중입니다.

## 설명
이번 변경은 마치 요리를 하기 전에 **[레시피를 먼저 확정(A)]**하고, 실제 요리할 때는 **[칼질 하나하나를 미리 설명하고 승인받는(B)]** 것과 같습니다. 요리가 끝난 후에는 **[주방을 정리하고 일지를 쓰는(C)]** 과정을 추가하여, 누가 봐도 어떤 과정으로 결과물이 나왔는지 알 수 있게 '철저한 기록'을 남기는 시스템을 구축한 것입니다.
