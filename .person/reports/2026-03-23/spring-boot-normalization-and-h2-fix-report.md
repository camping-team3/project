## 2. 주요 변경 사항 (Key Changes)

### 2.1 스프링 부트 및 Gradle 버전 정상화 (`build.gradle`)
- **이전**: 존재하지 않는 버전인 `4.0.3`으로 설정되어 스프링 부트의 자동 설정(Auto-configuration)이 제대로 작동하지 않았습니다.
- **변경**: 안정적인 최신 버전인 **`3.3.4`**로 수정하고, Gradle 의존성 관리 플러그인(`1.1.6`)을 적용하여 빌드 안정성을 확보했습니다.
- **H2 라이브러리**: 버전 충돌 방지를 위해 스프링 부트가 관리하는 버전을 사용하도록 의존성 설정을 최적화했습니다.

### 2.2 H2 콘솔 보안 및 인터셉터 차단 해결
- **SecurityConfig.java**: `PathRequest.toH2Console()`을 사용하여 H2 콘솔 경로에 대해 CSRF 검증을 무시하고, 보안 필터 체인에서 `ignoring()` 처리하여 완전한 접근을 허용했습니다.
- **WebMvcConfig.java**: 세션 체크를 수행하는 `LoginInterceptor`와 권한을 체크하는 `AdminInterceptor`에서 `/h2-console/**` 경로를 명시적으로 제외(`excludePathPatterns`)했습니다.

### 2.3 예외 처리 로직 및 진단 도구 강화
- **GlobalExceptionHandler.java**: 
  - 페이지를 찾을 수 없는 404 에러(`NoResourceFoundException`) 발생 시 무조건 로그인 폼으로 리다이렉트되던 현상을 해결했습니다. 이제는 404 전용 알림창을 띄워 사용자에게 정확한 상태를 전달합니다.
- **디버그 로그 추가**: `LoginInterceptor`, `AdminInterceptor`, `GlobalExceptionHandler`에 `[DEBUG]` 로그를 추가하여 요청 URI와 차단 원인을 실시간으로 확인할 수 있게 했습니다.

### 2.4 인코딩 설정 추가 (`application.properties`)
- UI에서 한글이 `???`로 출력되던 문제를 해결하기 위해 서버 및 템플릿 엔진(Mustache)의 기본 인코딩을 **UTF-8**로 강제 설정했습니다.

## 3. 검증 결과 (Validation)
- **빌드 성공**: `./gradlew classes` 명령을 통해 모든 컴파일 오류가 해결되었음을 확인했습니다.
- **접근성 확인**: `http://localhost:8080/h2-console` 접속 시 로그인 폼으로 튕기지 않고 H2 콘솔 로그인 화면이 정상적으로 나타납니다.
- **인코딩 확인**: 메인 화면 및 각 도메인 페이지의 한글 텍스트가 정상적으로 출력됩니다.

## 4. 비유를 통한 기술 설명 (Analogy)
이번 작업은 마치 **"잘못된 지도와 잠겨버린 관리자 전용 통로를 수리한 것"**과 같습니다.

- **스프링 부트 버전 수정**: 건물을 지을 때 설계도(버전)의 단위가 잘못되어 있어서(4.0.3), 배관이나 전기(자동 설정)가 하나도 연결되지 않던 상황을 **올바른 설계도(3.3.4)**로 교체하여 건물의 모든 기능이 정상적으로 작동하게 만든 것입니다.
- **H2 콘솔 접근성 개선**: 건물 관리실(H2 Console)로 가는 복도에 너무 많은 보안 경비원(Interceptor)이 배치되어 있어 정작 관리자가 들어가지 못하던 상황을 해결했습니다. 경비원들에게 **"관리실로 가는 사람은 검문하지 마라"**는 지침을 명확히 전달한 것입니다.
- **404 에러 리다이렉트 수정**: 손님이 길을 잃었을 때(404), 다짜고짜 **"신분증(로그인)부터 보여주세요"**라고 하던 불친절한 안내원을 **"죄송합니다, 그 길은 없는 길입니다"**라고 정확히 알려주는 친절한 안내원으로 바꾼 것과 같습니다.
