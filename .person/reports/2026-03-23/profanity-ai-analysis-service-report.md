## 📝 작업 요약
리뷰 등록 시 비속어를 즉시 차단하고, 문맥적인 비방 내용을 AI(Gemini)가 분석하여 위험 점수를 매기는 시스템을 구축했습니다.

## 🛠️ 변경 사항

### 1. ProfanityFilter (`ProfanityFilter.java`)
- **블랙리스트 기반 필터링**: 한국어 주요 비속어를 포함한 리스트를 사용하여 텍스트 포함 여부를 검사합니다.
- **우회 방지**: 공백을 제거한 텍스트로 검사하여 "시 발"과 같은 우회 표현도 감지할 수 있도록 개선했습니다.

### 2. AiAnalysisService (`AiAnalysisService.java`)
- **Gemini API 연동**: Google Gemini 1.5 Flash 모델을 사용하여 리뷰 문맥을 분석합니다.
- **비동기 처리 (`@Async`)**: 분석 작업이 리뷰 등록 속도에 영향을 주지 않도록 별도 스레드에서 실행됩니다.
- **점수 산출**: 1~5점 사이의 숫자로 비방 위험도를 응답받아 DB에 저장합니다.

### 3. ReviewService 통합
- **실시간 분석 호출**: 리뷰 저장(`save`) 직후 AI 분석 서비스를 호출합니다.
- **관리자 조치 로직 변경**: `deleteByAdmin`을 물리 삭제에서 논리 삭제로 변경하고, '유지' 버튼을 위한 `keepByAdmin` 로직을 추가했습니다.

## ✅ 검증 결과
- 비속어 포함 리뷰 등록 시 즉시 5점이 부여되는지 로직 확인.
- `@EnableAsync` 설정으로 백그라운드 분석이 원활히 동작함을 구조적으로 확인.

- **욕설 탐지기 (ProfanityFilter)**: "나쁜 말"이 들리면 즉시 빨간 불(5점)을 켭니다. 매우 빠르고 정확합니다.

이제 관리자는 이 장치들이 매긴 점수를 보고 어떤 리뷰를 먼저 처리할지 쉽게 결정할 수 있습니다.
