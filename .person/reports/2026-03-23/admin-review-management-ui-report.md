## 📝 작업 요약
관리자가 AI 분석 결과를 한눈에 확인하고, 부적절한 리뷰를 신속하게 처리할 수 있는 통합 관리 대시보드와 관련 기능을 구현했습니다.

## 🛠️ 변경 사항

### 1. 관리자 리뷰 목록 UI (`list.mustache`)
- **디자인 가이드 준수**: `design-rule.md`에 따라 `.fh-card`, `.fh-table` 및 브랜드 컬러(`--fh-primary`)를 적용했습니다.
- **AI 위험도 강조**: 위험도 3점 이상인 리뷰는 행 배경색을 붉은색으로 강조하여 관리자의 주의를 환기합니다.
- **상태 배지**: '검토대기', '유지됨', '삭제됨' 상태를 시각적인 배지로 구분했습니다.
- **SweetAlert2 연동**: 삭제 사유 입력 및 유지 확정 시 세련된 팝업 UI를 제공합니다.

### 2. 관리자 액션 API (`AdminController.java`)
- **조회 확장**: 정렬(위험도순/최신순) 및 검색 기능을 지원하도록 컨트롤러를 고도화했습니다.
- **[유지] API**: 리뷰를 '검토 완료' 상태로 변경하고 사용자 화면에 계속 노출합니다.
- **[삭제] API**: 관리자가 입력한 사유를 저장하며 논리 삭제 처리합니다. 삭제 시 실시간 평점 재계산 로직이 트리거됩니다.

## ✅ 검증 결과
- 위험도 3점 이상 리뷰가 빨간색으로 강조 표시되는지 확인.
- [삭제] 클릭 시 사유 입력창이 뜨고, 완료 후 목록에서 '삭제됨' 배지가 표시되는지 확인.
- [유지] 클릭 시 '유지됨' 배지로 변경되며 관리 버튼이 사라지는지 확인.

## 💡 비유로 설명하기
이번 작업은 **'리뷰 검문소'의 사무실과 업무용 컴퓨터를 설치한 것**과 같습니다.
- **모니터 (UI)**: 어떤 리뷰가 위험한지 AI가 미리 빨간 깃발을 꽂아둔 것을 큰 화면으로 보여줍니다.
- **삭제/유지 도장 (API)**: 관리자가 내용을 확인한 뒤 "이건 괜찮아(유지)" 혹은 "이건 안 돼(삭제)" 도장을 찍는 버튼입니다.
- **사유 메모 (Reason)**: 삭제 도장을 찍을 때 그냥 찍는 게 아니라 "왜 안 되는지" 메모를 남겨서 작성자가 납득할 수 있게 합니다.
이제 관리자는 복잡한 DB 수정 없이 클릭 몇 번으로 깨끗한 리뷰 게시판을 유지할 수 있게 되었습니다.
