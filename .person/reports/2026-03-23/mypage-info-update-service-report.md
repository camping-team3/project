# 회원 정보 수정 서비스 로직 구현 보고서

**Reporter**: Gemini CLI
**Date**: 2026-03-23
**Task Status**: Phase 3-2.5 (Task 2) 완료

## 📝 작업 요약
회원 정보를 실제 데이터베이스에 반영하고 검증하는 핵심 비즈니스 로직을 `UserService`에 구현했습니다.

## 🛠️ 변경 사항
1. **UserService.java**: 
   - `update(Long id, UserRequest.UpdateDTO request)` 메서드를 추가했습니다.
   - **보안 검증**: `passwordEncoder.matches()`를 사용하여 현재 비밀번호가 일치하는지 반드시 확인하도록 구현했습니다.
   - **선택적 비밀번호 변경**: 사용자가 새 비밀번호를 입력한 경우에만 암호화하여 업데이트하도록 처리했습니다.
   - **변경 감지**: `@Transactional`을 활용하여 엔티티의 상태 변화가 자동으로 DB에 반영되도록 설정했습니다.

## 🧪 검증 결과
- 현재 비밀번호가 틀릴 경우 `Exception400`이 발생하는 것을 로직상 확인했습니다.
- 새 비밀번호와 확인용 비밀번호가 불일치할 경우의 예외 처리를 완료했습니다.
- 이름, 이메일, 전화번호가 정상적으로 엔티티에 반영됨을 확인했습니다.

## 💡 비유로 설명하기
정보 수정 서비스 로직은 **'철저한 신분 확인을 거치는 은행원'**과 같습니다.
- 고객이 정보를 바꾸러 오면, 먼저 **'기존 비밀번호'**라는 신분증을 대조하여 본인인지 엄격히 확인합니다.
- 본인 확인이 완료된 후에야 요청한 대로 주소(이름/이메일 등)를 바꾸거나 열쇠(비밀번호)를 교체해 줍니다.
- 이 모든 과정이 끝나면 은행 장부에 최종적으로 기록이 남게 됩니다.
