# [보고서] 자바스크립트를 이용한 전역 금액 포맷팅(콤마) 적용

**Reporter**: parkcoding (Git user)
**Date**: 2026-03-23
**Status**: 완료 (Completed)

## 1. 작업 개요
사용자 및 관리자 예약 관련 모든 페이지에서 금액이 천 단위 콤마(,) 없이 출력되던 문제를 해결했습니다. 백엔드 DTO 수정 없이 자바스크립트를 활용하여 화면 렌더링 시점에 포맷팅을 수행하도록 구현했습니다.

## 2. 주요 변경 사항

### 2.1 프론트엔드 (Mustache & JS)
- **금액 출력부 구조 변경**: 모든 금액 변수(`{{totalPrice}}`, `{{pricePerNight}}` 등)를 `<span class="format-price">...</span>` 태그로 감싸 식별이 가능하도록 했습니다.
- **공통 포맷팅 스크립트 추가**: 대상 페이지 하단에 `.format-price` 클래스를 가진 모든 요소의 텍스트를 `toLocaleString()`을 사용하여 콤마가 포함된 형식으로 변환하는 스크립트를 삽입했습니다.
- **적용 대상 파일**:
    - `reservation/new.mustache` (사이트 목록)
    - `reservation/payment.mustache` (결제 예정 내역 및 버튼)
    - `reservation/complete.mustache` (예약 완료 상세)
    - `admin/reservation/list.mustache` (관리자 목록 테이블)
    - `admin/reservation/detail.mustache` (관리자 상세 정보)
    - `admin/reservation/cancel-detail.mustache` (취소 요청 상세)

## 3. 구현 방식의 장점
- **백엔드 무결성 유지**: Java DTO나 서비스 로직을 전혀 수정하지 않아 기존 비즈니스 로직에 영향을 주지 않습니다.
- **일관성 확보**: 기존 `new.mustache`의 예약 요약(JS 계산 영역)에서 사용하던 방식과 통일하여 기술적 일관성을 맞췄습니다.
- **확장성**: 추후 통화 단위 기호를 변경하거나 다른 포맷팅 규칙을 적용할 때 자바스크립트 한 곳만 수정하면 됩니다.

## 4. 종합 결론
이번 작업을 통해 모든 예약 프로세스 및 관리 화면에서 금액이 `100,000원`과 같이 가독성 있게 표시됩니다. 이는 사용자 편의성을 높이고 관리자의 업무 효율을 향상시키는 효과를 가져올 것입니다.

**비유를 통한 설명**: 
상대방에게 돈을 줄 때 그냥 뭉칫돈으로 주던 것을, 이제는 **'은행용 고무줄(콤마)'**로 만 원 단위로 묶어서 주는 것과 같습니다. 받는 사람은 이제 한눈에 얼마인지 세어보지 않아도 바로 알 수 있게 되었습니다.
