# 회원 정보 수정 DTO 및 엔티티 업데이트 보고서

**Reporter**: Gemini CLI
**Date**: 2026-03-23
**Task Status**: Phase 3-2.5 (Task 1) 완료

## 📝 작업 요약
마이페이지에서 회원 정보를 수정하기 위해 필요한 데이터 전송 객체(DTO)를 정의하고, 실제 DB에 반영하기 위한 엔티티의 업데이트 메서드를 구현했습니다.

## 🛠️ 변경 사항
1. **UserRequest.java**: 
   - `UpdateDTO` 정적 클래스를 추가했습니다.
   - 이름, 이메일, 전화번호와 함께 보안을 위한 현재 비밀번호, 변경할 새 비밀번호 필드를 포함합니다.
2. **User.java**: 
   - JPA의 변경 감지 기능을 활용할 수 있도록 `updateInfo`와 `updatePassword` 메서드를 추가했습니다.
   - 직접적인 필드 접근을 지양하고 의미 있는 메서드를 통해 상태를 변경하도록 설계했습니다.

## 🧪 검증 결과
- `UserRequest.UpdateDTO` 객체가 정상적으로 선언되어 컨트롤러에서 활용 가능함을 확인했습니다.
- `User` 엔티티의 메서드가 의도한 필드들을 정확히 업데이트함을 코드 레벨에서 검토했습니다.

## 💡 비유로 설명하기
회원 정보를 수정하는 것은 **'오래된 회원 카드를 갱신하는 것'**과 같습니다. 
- `UpdateDTO`는 사용자가 새로 작성해 낸 **'변경 신청서'**이며, 
- `User` 엔티티의 메서드는 신청서를 보고 실제 **'장부의 내용을 고쳐 쓰는 작업'**입니다. 
이제 신청서도 준비되었고, 장부를 고쳐 쓸 펜도 준비되었습니다!
