## 1. 작업 요약
사용자가 마이페이지에서 본인의 예약을 취소 요청할 수 있는 기능을 구현했습니다. 단순 삭제가 아닌 '취소 요청(CANCEL_REQ)' 상태로 전환하여 관리자의 최종 승인을 거치도록 설계되었습니다.

## 2. 변경 사항

### 2.1 백엔드 (Java)
- **`ReservationRequest.java`**: 취소 사유 및 환불 계좌 정보를 수집하기 위한 `CancelDTO` 추가
- **`ReservationResponse.java`**: 취소 요청 완료 화면에 표시할 데이터를 담는 `CancelDoneDTO` 추가
- **`ReservationService.java`**: 
  - `requestCancel`: 예약 소유권 검증, 이용일 3일 전 체크 규칙 적용, 예약 상태 변경(`CANCEL_REQ`), `ReservationCancelRequest` 이력 저장 로직 구현
  - `getCancelDoneDetails`: 최근 취소 요청 정보 조회 로직 추가
- **`ReservationController.java`**: 취소 폼 조회, 취소 요청 처리, 완료 페이지 리다이렉트 엔드포인트 구현

### 2.2 프론트엔드 (Mustache)
- **`reservation-cancel.mustache`**: 취소 사유와 환불 계좌(은행, 계좌번호, 예금주)를 입력받는 인터랙티브 폼 구현
- **`reservation-cancel-done.mustache`**: 요청 접수 완료를 알리고 신청 정보를 요약해서 보여주는 결과 페이지 구현

## 3. 검증 결과 및 비즈니스 규칙 적용
- **소유권 검증**: 본인의 예약이 아닌 경우 취소 요청이 불가하도록 처리 (`Exception403`)
- **기한 검증**: 이용일(체크인) 기준 3일 전까지만 온라인 취소가 가능하도록 제한 (`Exception400`)
- **데이터 정합성**: 취소 요청 시 원본 예약의 상태가 즉시 `CANCEL_REQ`로 변경되어 타 메뉴에서의 중복 조작을 방지함

## 4. 비유로 설명하는 이번 기능
이번 기능은 **"음식점 예약 취소 전화"**와 비슷합니다. 손님이 직접 예약을 지우는 것이 아니라, "이러이러한 사유로 취소하고 싶어요"라고 요청을 남기고 환불받을 계좌를 알려주는 단계입니다. 식당 주인(관리자)은 이 요청을 보고 위약금 규정을 확인한 뒤 최종적으로 "네, 취소 처리해드렸습니다"라고 승인해주는 과정의 첫 단추를 끼운 것입니다.
