# 결제 시스템 환경 구축 및 Mock 구조 설계 리포트
Date: 2026-03-20
Reporter: Git user.name
Task: Phase 3-2.1 Task 1

## 1. 작업 요약
포트원(PortOne) V2 연동을 위한 기초 환경을 구축하고, 실제 외부 API 없이도 개발이 가능하도록 인터페이스 기반의 Mock 서비스 구조를 설계했습니다.

## 2. 변경 사항
- **설정 추가**: `application.properties`에 포트원 API 관련 환경 변수 구조 정의
- **설정 매핑**: `PortOneProperties` 클래스를 통해 설정값들을 자바 객체로 캡슐화
- **인터페이스 설계**: `PaymentService`를 도입하여 결제 엔진(PortOne vs Mock)의 유연한 교체 가능성 확보
- **Mock 구현**: `MockPaymentService`를 통해 로컬 환경에서 결제 검증 및 취소 시뮬레이션 가능하게 구현
- **프로파일 분리**: `@Profile` 어노테이션을 사용하여 운영(`prod`)과 비운영 환경의 빈 주입을 자동화

## 3. 검증 결과
- 빌드 및 컴파일 체크 완료.
- `@ConfigurationProperties` 매핑 구조 확인.
- 비운영 환경(`!prod`)에서 `MockPaymentService`가 빈으로 등록되는 아키텍처 확립.

## 4. 비유로 설명하기
이 작업은 마치 **'전기차 충전 시스템을 만들 때, 실제 충전기를 연결하기 전에 전기가 들어오는 흉내를 내는 모의 시험기를 먼저 설치한 것'**과 같습니다. 나중에 실제 충전기(포트원)가 준비되면 코드 수정 없이 스위치(프로파일)만 바꾸면 바로 작동하게 됩니다.
