## 5. 💡 기술 공부방: AJAX와 await fetch 이해하기

이번 작업의 핵심인 **비동기 통신(AJAX)**에 대해 알아봅시다.

### 5.1 AJAX란 무엇인가요?
**AJAX**는 "Asynchronous JavaScript And XML"의 약자로, 쉽게 말해 **"페이지 전체를 새로고침하지 않고도, 백그라운드에서 서버와 데이터를 주고받는 기술"**입니다.
- **일반적인 방식**: 버튼을 누르면 화면이 하얗게 변했다가 새로운 페이지가 뜹니다 (전체 새로고침).
- **AJAX 방식**: 화면은 그대로 있는데, 특정 부분의 데이터만 슥- 바뀌거나 추가됩니다 (부분 업데이트).

### 5.2 fetch와 async/await은 무엇인가요?
서버에 데이터를 요청할 때 사용하는 현대적인 도구가 바로 `fetch`입니다. 그리고 그 요청이 끝날 때까지 기다려주는 문법이 `async/await`입니다.

#### [핵심 코드 예시]
```javascript
// 1. async: "이 함수 안에는 기다려야(await) 하는 작업이 있어요!"라고 선언합니다.
async function fetchSites() {
    // 2. fetch: 서버에 "이 주소로 가서 데이터 좀 가져와줘!"라고 심부름을 보냅니다.
    // 3. await: "심부름꾼이 돌아올 때까지 다음 줄로 가지 말고 잠깐 기다려!"라고 명령합니다.
    const response = await fetch('/api/url');

    // 4. .json(): 서버가 준 데이터를 자바스크립트가 이해할 수 있는 객체(Object)로 변환합니다.
    // 이것도 시간이 걸리는 일이라 await로 기다려줍니다.
    const sites = await response.json();

    // 5. 이제 가져온 데이터를 화면에 그리는 작업을 합니다.
    console.log(sites); 
}
```

### 5.3 왜 await을 쓰나요?
서버에 데이터를 요청하는 일은 아주 짧은 시간이지만 "기다림"이 필요합니다. 
만약 `await` 없이 코드를 짜면, 심부름꾼이 아직 도착하지도 않았는데 "데이터 보여줘!"라고 다음 코드를 실행해버려서 에러가 납니다. 
`await`는 **"응답이 올 때까지 여기서 딱 대기했다가, 데이터가 도착하면 안전하게 다음 작업을 해!"**라고 순서를 정해주는 아주 고마운 문법입니다.
