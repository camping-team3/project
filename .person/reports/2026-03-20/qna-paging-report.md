# 📝 리포트: QnA 커뮤니티 페이징 기능 구현 및 데이터 확장

**날짜**: 2026-03-20
**Reporter**: KimNaKim
**작업 단계**: Phase 3 - QnA 페이징 기능 구현

## 1. 작업 요약
- QnA 게시판에 페이지당 5개 노출, 최신순 정렬의 서버 사이드 페이징을 구현하였습니다.
- 기존의 '답변 상태' 필터와 페이징이 유기적으로 연동되도록 고도화하였습니다.
- 테스트를 위한 대량의 더미 데이터를 추가하고 SQL 오류를 수정하였습니다.

## 2. 변경 사항

### 2.1 백엔드 고도화 (`QnaRepository`, `QnaService`)
- **Pageable 도입**: 리스트 반환 방식에서 `Page` 객체 반환 방식으로 확장하여 서버 부하를 줄였습니다.
- **PageDTO 신설**: 현재 페이지, 총 페이지, 이전/다음 존재 여부, 페이지 번호 리스트를 포함하는 전용 DTO를 구축하였습니다.

### 2.2 컨트롤러 연동 (`QnaController`, `AdminController`)
- **파라미터 처리**: `@RequestParam`을 통해 페이지 번호를 수신하고, 5개씩 ID 역순(최신순)으로 정렬하도록 설정하였습니다.
- **일관성 확보**: 사용자용 QnA 목록과 관리자용 QnA 관리 페이지 모두에 동일한 페이징 로직을 적용하였습니다.

### 2.3 프론트엔드 UI (`list.mustache`)
- **동적 렌더링**: Mustache 루프를 이용해 하단 페이지 번호를 동적으로 생성하였습니다.
- **필터 유지**: '전체/대기/완료' 필터 상태가 페이지 이동 시에도 쿼리 스트링(`status=...`)을 통해 유지되도록 구현하였습니다.

### 2.4 데이터베이스 (`data.sql`)
- **더미 데이터 확장**: 총 15개의 QnA 데이터를 확보하여 다중 페이지 테스트 환경을 조성하였습니다.
- **오류 수정**: Enum에 정의되지 않은 값('SAFETY')을 'ETC'로 일괄 수정하여 DB 초기화 오류를 해결하였습니다.

## 3. 검증 결과
- **정렬 및 노출**: 페이지당 5개의 카드가 정확히 노출되며, 최신글이 상단에 위치함을 확인하였습니다.
- **상태 유지**: 특정 필터를 선택한 상태에서 페이지를 넘겨도 필터 조건이 풀리지 않고 정상 동작함을 확인하였습니다.
- **관리자 연동**: 관리자 페이지에서도 페이징 번호 클릭 시 해당 데이터가 정확히 로드됨을 확인하였습니다.

## 4. 어려운 내용 설명 (비유로 설명하기)
### 🎫 줄 서서 기다리는 손님들을 위한 번호표 (Paging)
비유하자면, 캠핑장 안내소에 손님이 너무 많이 몰려 입구가 꽉 막혀 있던 상황이었습니다. 이전에는 모든 손님을 한꺼번에 입장시키려 해서 안내원도 손님도 모두 힘들었지만, 이제는 **"5명씩 끊어서 번호표"**를 나눠주기 시작했습니다. 1번 그룹이 들어가면 2번 그룹이 대기하고, 손님이 "나는 가족끼리 온 팀이야(필터)"라고 말하면 그 팀들 중에서만 다시 번호표를 매겨 순서대로 입장시키는 똑똑한 안내 시스템을 구축한 것입니다!

## 5. 💡 기술 공부방: Spring Data JPA 페이징 (Pageable)

### 5.1 Pageable이란?
스프링 데이터 JPA에서 제공하는 **페이징과 정렬 정보를 담는 인터페이스**입니다.
- `PageRequest.of(page, size, sort)`: 몇 페이지를, 몇 개씩, 어떤 순서로 가져올지 정의합니다.

### 5.2 Page vs List
- **List**: 조건에 맞는 모든 데이터를 한꺼번에 가져옵니다. 데이터가 1만 개라면 성능에 치명적입니다.
- **Page**: 딱 필요한 만큼만 가져오고, **"전체 데이터는 몇 개인지", "다음 페이지가 있는지"** 같은 정보까지 덤으로 알려줍니다.

### 5.3 왜 ID 역순(descending)으로 정렬하나요?
데이터베이스에서 ID는 보통 생성된 순서대로 커집니다. 따라서 **ID를 큰 숫자부터(내림차순)** 가져오면, 사용자가 가장 궁금해하는 **"가장 최근에 올라온 글"**을 제일 먼저 보여줄 수 있습니다.
