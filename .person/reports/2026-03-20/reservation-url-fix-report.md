# 📝 리포트: 예약 관련 URL 구조 통일 및 리다이렉트 버그 수정

**날짜**: 2026-03-20
**Reporter**: parkcoding
**작업 단계**: Phase 3 - Task 3-3 보완 (버그 수정 및 리팩토링)

## 1. 작업 요약
- 마이페이지 내 예약 관련 URL 패턴을 `/mypage/reservations/{id}/액션` 구조로 통일하였습니다.
- 로그인 상태임에도 예약 변경 페이지 진입 시 로그인 페이지로 리다이렉트되던 버그를 해결하였습니다.

## 2. 변경 사항

### 2.1 컨트롤러 (`ReservationController.java`)
- 예약 상세, 변경 폼, 변경 요청(POST), 변경 완료 URL을 `/mypage/reservations/{id}/...` 구조로 변경하였습니다.
- `@PathVariable("id")`를 활용하여 경로 변수를 일관되게 처리하도록 수정하였습니다.

### 2.2 뷰 템플릿 (`reservations.mustache`, `reservation-change.mustache`)
- 예약 목록의 모든 링크(상세, 변경, 취소)를 새로운 URL 패턴에 맞춰 업데이트하였습니다.
- 예약 변경 신청 폼의 `action` URL을 복수형 및 경로 변수 기반으로 수정하였습니다.

## 3. 검증 결과
- **버그 해결 확인**: 예약 목록에서 '예약 변경' 클릭 시 로그인 페이지로 튕기지 않고 정상적으로 변경 신청 폼이 노출됨을 확인하였습니다.
- **일관성 확보**: 모든 마이페이지 예약 관련 요청이 `/mypage/reservations`라는 공통 부모 경로 아래에서 리소스 ID(`{id}`)를 포함하는 표준화된 RESTful 스타일의 URL로 동작합니다.

## 4. 어려운 내용 설명 (비유로 설명하기)
### 📮 주소 체계 통일 (URL Refactoring)
비유하자면, 우리 마을의 우편함 주소가 어떤 집은 `마을/집/1/편지함`이고 어떤 집은 `마을/편지함/1`으로 제각각이어서 집배원(Spring Framework)이 길을 잃고 엉뚱한 곳(로그인 페이지)으로 안내하던 상황이었습니다. 
이번 작업을 통해 모든 집의 주소를 `마을/예약들/집번호/편지함`이라는 일정한 규칙으로 통일했습니다. 이제 집배원이 헷갈리지 않고 정확하게 예약 변경 신청서나 상세 내역을 배달할 수 있게 되었습니다!
