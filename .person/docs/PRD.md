# 🏕️ 캠핑장 예약 시스템 PRD (Product Requirements Document)

## 1. 프로젝트 개요 (Project Overview)

본 프로젝트는 **단일 캠핑장 사업장**을 위한 독립적인 풀스택 웹 서비스 구축을 목표로 합니다. 고객은 실시간으로 캠핑장 사이트의 빈자리를 확인하여 결제 및 예약을 진행하고, 관리자는 예약 현황, 환불 처리, 홈페이지 콘텐츠(공지사항, 갤러리, Q&A)를 통합 관리자 모드에서 독립적으로 제어합니다.

## 2. 대상 사용자 (Target Audience)

- **고객 (Customer):** 회원가입 및 로그인 후 캠핑장을 예약하고, 본인의 예약/결제 내역을 관리하며, 리뷰 및 Q&A를 남기는 실사용자.
- **관리자 (Admin):** 캠핑장 운영자. 예약 및 취소 현황 모니터링, 취소 요청에 대한 위약금 확인 및 최종 환불 승인, 공지사항/갤러리 등 사이트 전반의 데이터를 관리.

## 3. 핵심 요구사항 (Key Features & Requirements)

### 3.1. 인증 및 회원 관리 (Authentication)

- **회원 전용 서비스:** 비회원 예약은 지원하지 않으며, 예약, 리뷰 작성, Q&A 작성 등 주요 기능은 모두 로그인 후에만 이용 가능합니다.
- **권한 분리:** 일반 회원과 관리자의 권한을 명확히 분리하여, 비인가자의 관리자 페이지 접근을 원천 차단합니다.

### 3.2. 실시간 예약 시스템 (Reservation Flow)

- **즉시 확정 시스템:** 고객이 날짜/인원을 선택하고 카드/간편 결제를 완료하면 예약이 '즉시 확정'됩니다. (무통장 입금 후 수동 확정 방식 배제)
- **오버부킹 방지 (선점 Lock 시스템):** [결제하기] 화면에 진입하는 순간 해당 캠핑 사이트와 일정을 5분간 임시 선점(Lock)하여, 다른 사용자의 중복 결제 시도를 시스템적으로 차단합니다. 5분 내 결제 미완료 시 Lock은 자동 해제됩니다.

### 3.3. 예약 취소 및 환불 제어 (Cancellation Flow)

- **관리자 수동 승인:** 고객이 마이페이지에서 취소 '요청'을 하더라도 즉시 환불되지 않습니다.
- **안전한 운영:** 관리자가 대시보드에서 해당 취소 요청을 확인하고, 이용일까지 남은 기간에 따른 위약금 규정을 검토한 뒤 '최종 승인' 버튼을 눌러야만 실제 환불 연동 및 예약 취소 처리가 완료됩니다.

### 3.4. 커뮤니티 및 소통 (Community)

- **Q&A 게시판:** 고객은 질문을 남길 수 있고, 관리자는 이에 답변을 답니다. 답변이 완료된 게시글은 고객이 임의로 수정/삭제할 수 없도록 무결성을 유지합니다.
- **검증된 리뷰:** '이용 완료' 상태의 예약 건을 보유한 고객만 리뷰(별점, 사진 첨부)를 작성할 수 있습니다.

### 3.5. 관리자 대시보드 (Admin Dashboard)

- **통합 운영:** 시즌(성수기/비수기)별 요금 설정, 예약 상태(대기/확정/취소) 변경, 사이트 메인에 노출될 포토 갤러리 및 공지사항 등록 등을 하나의 대시보드에서 처리합니다.

## 4. 시스템 아키텍처 및 기술 스택 (System Architecture)

본 프로젝트는 API 분리형(SPA)이 아닌, 견고한 **서버 사이드 렌더링(SSR)** 아키텍처를 채택합니다.

- **Backend:** Spring Boot (Java)
- **Database:** MySQL (예약 트랜잭션 및 Lock 테이블 관리)
- **Template Engine:** Mustache (컨트롤러에서 Model 객체를 바인딩하여 HTML 화면 렌더링)
- **Security:** `HttpSession` (상태 유지) + Spring `HandlerInterceptor` (URL 라우팅 기반의 강력한 권한 인가 필터)

## 5. 통신 및 개발 원칙 (Development Constraints)

- **Form Submit 기반 상태 변경:** 데이터 조회는 `GET`, 생성/수정/삭제 등 상태 변경은 HTML `<form method="POST">`와 PRG(Post-Redirect-Get) 패턴을 사용합니다.
- **제한적 비동기 통신:** AJAX 처리는 페이지 전체 리로딩이 UX를 크게 해치는 극히 예외적인 상황(예: 실시간 캘린더 예약 가능 여부 렌더링, 회원가입 시 이메일 중복 체크 등)에만 제한적으로 허용합니다.

## 6. 개발 범위 외 (Out of Scope)

- 비회원 예약 및 결제 기능
- 무통장 입금 확인 기반의 수동 예약 확정 프로세스
- 관리자 승인 절차를 생략한 시스템 자동 즉시 환불 로직
