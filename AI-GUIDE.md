# AI Guide

> 개발자의 역할은 코드를 직접 짜는 것이 아니라, **workflow를 잘 설계하는 것**이다.
> 명확한 workflow를 작성하고 AI에게 지시하는 것 — 이것이 앞으로의 개발 방식이다.
> 코드는 AI가 만든다. 개발자는 **무엇을, 왜, 어떤 순서로** 만들지를 정의한다.

---

## 0. 세션 초기화 루틴 (최우선 실행)

새로운 세션이 시작되면 어떠한 사용자 요청보다 먼저 다음 단계를 **자동으로** 수행한다. 이는 에이전트가 바뀔 때마다 사용자가 중복된 설명을 하지 않도록 하기 위함이다.

1.  **상태 로드**: `.ai/AI_CONTEXT.md`를 즉시 읽고 현재 프로젝트의 **마일스톤(1번 항목)**과 **최근 업데이트 내역(5번 항목)**을 완벽히 파악한다.
2.  **무결성 검사**: `AI_CONTEXT.md`의 **2. 프로젝트 맵**에 명시된 파일 명세와 실제 디렉토리의 파일들을 대조한다. 만약 명세와 실제 구조가 다르다면(파일 누락 등) 즉시 사용자에게 보고하고 동기화를 제안한다.
3.  **초기화 보고**: 위 과정이 끝나면 사용자에게 다음과 같은 형식으로 첫 인사를 건네며 업무를 시작한다.
    > "세션 초기화를 완료했습니다. 현재 **[마일스톤 명칭]** 단계이며, 진척도는 **[XX%]**입니다. 마지막 작업은 **[최근 작업 내용]**이었습니다. 오늘은 이어서 **[다음 목표]**를 진행할까요?"

## 1. 자동 동기화 (Skill Sync)

세션 초기화 직후, `./ai/skills` 폴더의 스킬들을 읽어 아래 Skill 테이블과 동기화한다.

| 분류 | 이름 | 트리거/용도 | 주요 능력 |
| :--- | :--- | :--- | :--- |
| 분석 | deep-interview | /deep-interview, 요구사항 정리, 기능 정의 요청 시 | 소크라테스식 질문을 통해 모호한 요구사항을 구조화된 명세로 변환 |
| 분석 | plan | /plan, 기능 개발, 코드 수정 요청 시 | 코드 작성 전 '선 계획, 후 승인, 사후 보고' 프로세스 강제 |

## 2. 참조 규칙 (Memory Load)

코드 생성/수정 시 아래 파일을 읽고 규칙을 반드시 준수한다.

- **컨벤션**: `.ai/rules/common-rule.md`
- **스킬상세**: `.ai/skills/{스킬명}/SKILL.md`
- **비즈니스**: `.ai/rules/business-rule.md`
- **아키텍처** : `.ai/AI-CONTEXT.md`
